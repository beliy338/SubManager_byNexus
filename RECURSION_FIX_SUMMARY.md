## 🔍 Техническая информация

### Причина ошибки

Старая политика RLS содержала подзапрос к той же таблице:

```sql
-- ❌ НЕПРАВИЛЬНО - вызывает рекурсию
parent_message_id IN (
    SELECT id FROM support_messages WHERE user_id = auth.uid()
)
```

PostgreSQL пытался применить политику RLS к подзапросу, что создавало бесконечную рекурсию.

### Решение

Создана функция-помощник с `SECURITY DEFINER`:

```sql
-- ✅ ПРАВИЛЬНО - без рекурсии
CREATE FUNCTION is_user_parent_message(parent_id UUID, user_uuid UUID)
RETURNS BOOLEAN
LANGUAGE plpgsql
SECURITY DEFINER -- обходит RLS безопасно
AS $$
BEGIN
    RETURN EXISTS (
        SELECT 1 
        FROM support_messages 
        WHERE id = parent_id AND user_id = user_uuid
    );
END;
$$;
```

Теперь политика использует функцию вместо подзапроса:

```sql
-- Новая политика без рекурсии
is_user_parent_message(parent_message_id, auth.uid())
```
