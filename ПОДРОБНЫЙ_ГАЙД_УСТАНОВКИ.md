## ❓ Частые вопросы и решения

### ❌ Сервисы не появляются в приложении

**Решение:**
1. Обновите страницу (Ctrl+F5)
2. Проверьте, что вы вошли в систему
3. Откройте консоль браузера (F12) и проверьте ошибки
4. Убедитесь, что SQL скрипт выполнился успешно

---

### ❌ Ошибка "permission denied" при выполнении SQL

**Решение:**
1. Убедитесь, что вы вошли как владелец
2. Email должен быть: max.sokolvp@gmail.com или belovodvadim@gmail.com
3. Проверьте права доступа к таблице services
4. Попробуйте выйти и войти заново

---

### ❌ Сервисы добавились дважды

**Решение:**
1. Не переживайте! Скрипт использует `ON CONFLICT DO NOTHING`
2. Дубликаты не должны были появиться
3. Если всё-таки есть дубликаты, удалите их через Table Editor:
```sql
-- Найти дубликаты
SELECT name, COUNT(*)
FROM public.services
GROUP BY name
HAVING COUNT(*) > 1;

-- Удалить дубликаты (оставить только один)
DELETE FROM public.services a
USING public.services b
WHERE a.id < b.id
AND a.name = b.name;
```

---

### ❌ Категории "Развлечения" и "Шопинг" не видны в фильтре

**Решение:**
1. Обновите страницу приложения (Ctrl+F5)
2. Проверьте, что сервисы добавились с правильными категориями
3. Категории должны быть в файле `/src/app/components/SelectServiceModal.tsx`
4. Если проблема сохраняется, проверьте консоль браузера

---

### ❌ Цены отображаются в долларах вместо рублей

**Решение:**
1. Откройте Settings в приложении
2. Выберите валюту **RUB (₽)**
3. Приложение автоматически пересчитает все суммы
4. Обновите страницу

---

### ❌ Ошибка 403 при добавлении подписки

**Решение:**
1. Это нормально! Игнорируйте ошибку 403
2. Она связана с Edge Function, которая не используется
3. Приложение работает через Supabase SDK
4. Подписки должны добавляться нормально
