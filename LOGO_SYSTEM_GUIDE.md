# Руководство по единой системе отображения логотипов

**Дата создания:** 16 марта 2026  
**Статус:** ✅ Реализовано и активно

---

## 📋 Обзор

Создана единая масштабируемая система отображения логотипов сервисов через компонент `ServiceLogo`, который автоматически обрабатывает все сценарии отображения логотипов на всех страницах приложения.

---

## 🎯 Решённые проблемы

1. ✅ Логотипы теперь отображаются единообразно на всех страницах
2. ✅ Автоматический fallback для сервисов без логотипа (первая буква на градиентном фоне)
3. ✅ Легко масштабируется для новых сервисов
4. ✅ Поддержка разных размеров и вариантов отображения
5. ✅ Адаптация под светлую/тёмную темы

---

## 📦 Компонент ServiceLogo

### Расположение
`/src/app/components/ServiceLogo.tsx`

### Использование

```tsx
import { ServiceLogo } from '../components/ServiceLogo';

// Базовое использование
<ServiceLogo logo={subscription.logo} name={subscription.name} />

// С указанием размера
<ServiceLogo logo={service.icon} name={service.name} size="lg" />

// С дополнительными стилями
<ServiceLogo 
  logo={sub.logo} 
  name={sub.name} 
  size="md" 
  variant="rounded"
  className="shadow-md"
/>
```

### Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `logo` | `string \| undefined` | - | URL логотипа сервиса |
| `name` | `string` | - | Название сервиса (обязательно) |
| `size` | `'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'` | Размер логотипа |
| `variant` | `'square' \| 'rounded'` | `'rounded'` | Форма логотипа |
| `className` | `string` | `''` | Дополнительные CSS классы |

### Размеры

- **sm**: 8×8 (32px) - для мелких элементов
- **md**: 10×10 (40px) - стандартный размер для списков
- **lg**: 12×12 (48px) - для карточек и основных элементов
- **xl**: 16×16 (64px) - для детальных страниц

## 🎨 Визуальное поведение

### Если логотип есть (`logo` задан):
```tsx
<div className="w-10 h-10 rounded-lg overflow-hidden border border-border">
  <img src={logo} alt={name} className="w-full h-full object-cover" />
</div>
```

### Если логотипа нет (`logo` не задан):
```tsx
<div className="w-10 h-10 rounded-lg bg-gradient-to-br from-primary/20 to-accent/20 flex items-center justify-center font-bold">
  S
</div>
```

- Отображается первая буква названия сервиса
- Градиентный фон автоматически адаптируется к теме (оранжево-чёрный или светлый)
- Буква всегда uppercase

## 🚀 Добавление нового сервиса

### Для владельцев (через UI):

1. Зайти на страницу **Подписки** (Subscriptions)
2. Нажать **"Добавить сервис"**
3. Заполнить:
   - Название
   - Категорию (см. синхронизированный список ниже)
   - Описание
   - URL логотипа (рекомендуется квадратное изображение)
   - Тарифные планы
4. Логотип автоматически подхватится везде через `ServiceLogo`

### Для разработчиков:

Никаких изменений в коде не требуется! Просто добавьте сервис в базу данных:

```sql
INSERT INTO services (name, category, icon, description, is_popular, pricing_plans)
VALUES (
  'Новый Сервис',
  'Музыка',
  'https://example.com/logo.png',
  'Описание сервиса',
  false,
  '[{"id":"plan1","name":"Базовый","price":299,"currency":"RUB","billingCycle":"monthly"}]'
);
```

## 🔄 Миграция существующего кода

### Было (старый подход):
```tsx
{sub.logo ? (
  <div className="w-10 h-10 rounded-lg overflow-hidden border border-border flex-shrink-0">
    <img src={sub.logo} alt={sub.name} className="w-full h-full object-cover" />
  </div>
) : (
  <div className="w-10 h-10 rounded-lg bg-gradient-to-br from-primary/20 to-accent/20 flex items-center justify-center text-sm font-bold">
    {sub.name.charAt(0)}
  </div>
)}
```

### Стало (новый подход):
```tsx
<ServiceLogo logo={sub.logo} name={sub.name} size="md" />
```

**Экономия:** -10 строк кода на каждое использование!

## 🎯 Рекомендации

### ✅ Хорошо:
- Использовать квадратные логотипы (1:1)
- Рекомендуемый размер: 512×512px или больше
- Формат: PNG с прозрачностью или JPG
- Хостить на надёжных CDN (например, через Supabase Storage)

### ❌ Избегать:
- Прямоугольные логотипы (будут обрезаны)
- Логотипы меньше 128×128px (будут размыты)
- Логотипы со сложными деталями (плохо выглядят в малых размерах)

## 🐛 Устранение неполадок

### Проблема: Логотип не отображается

**Решение:**
1. Проверьте URL логотипа в базе данных
2. Убедитесь, что URL доступен публично
3. Проверьте CORS настройки сервера изображений
4. Fallback сработает автоматически (первая буква)

### Проблема: Логотип растянут/искажён

**Решение:**
- Используйте квадратные изображения
- Проверьте, что в оригинале соотношение сторон 1:1
