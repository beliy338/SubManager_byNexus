# 🔧 Исправление ошибки "Failed to fetch" при регистрации

## ❌ Проблема
При попытке создать новый аккаунт появлялась ошибка:
```
Signup request failed: TypeError: Failed to fetch
Signup error: TypeError: Failed to fetch
```

## ✅ Причина
Приложение пыталось обратиться к серверным функциям Supabase (Edge Functions), которые еще не были развернуты на сервере. Код регистрации использовал несуществующий API endpoint `/make-server-076c1030/signup`.

## 🛠️ Решение

### 1. Использование Supabase Auth напрямую
Вместо серверной функции теперь используется встроенный метод аутентификации:
```typescript
await supabase.auth.signUp({
  email,
  password,
  options: { data: { name } }
});
```

### 2. Временное хранение в localStorage
Все данные (подписки, настройки) сохраняются локально в браузере до момента развертывания Edge Functions.

### 3. Закомментированы API вызовы
В файле `/src/app/contexts/AppContext.tsx` все обращения к серверу закомментированы с пометкой `TEMPORARY`.

## 📁 Измененные файлы

1. **`/src/app/pages/Signup.tsx`**
   - Удален вызов `api.signup()`
   - Добавлен прямой вызов `supabase.auth.signUp()`
   - Обновлено информационное сообщение

2. **`/src/app/contexts/AppContext.tsx`**
   - Закомментирован импорт `api`
   - Все CRUD операции переведены на localStorage
   - Добавлены комментарии для будущей миграции

3. **`/SUPABASE_AUTH_FIX.md`** (создан)

4. **`/ИСПРАВЛЕНИЕ_ОШИБОК.md`** (этот файл)
