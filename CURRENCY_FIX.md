# Исправление двойной конвертации валют

## Проблема
При выборе сервиса из базы данных (например, Telegram Premium за 300 рублей) и установке валюты пользователя на доллары происходила **двойная конвертация**:

1. **При добавлении подписки** цена конвертировалась из RUB в USD (300 / 80 = 3.75) и сохранялась в БД как 3.75
2. **При отображении** эта уже конвертированная цена (3.75) снова конвертировалась (3.75 / 80 = 0.047), что давало неверный результат

## Решение
Все цены теперь **хранятся только в рублях (RUB)** в базе данных, а конвертация происходит **только при отображении**.

### Курсы валют
- 80 RUB = 1 USD
- 95 RUB = 1 EUR
- 12 RUB = 1 CNY
- 27 RUB = 1 BYN

## Изменённые файлы

### 1. `/src/app/contexts/AppContext.tsx`
**Добавлена функция `convertToRub`:**
```typescript
const convertToRub = (price: number, fromCurrency: string = 'RUB'): number => {
  if (fromCurrency === 'RUB') return price;
  
  // Convert to RUB
  return price * currencyRates[fromCurrency];
};
```

Эта функция конвертирует цену из любой валюты в рубли для сохранения в БД.

### 2. `/src/app/components/SelectServiceModal.tsx`
**Убрана конвертация при сохранении:**
```typescript
// Было:
const convertedPrice = convertPrice(selectedPlan.price, selectedPlan.currency);
await addSubscription({
  price: convertedPrice, // ❌ Сохранялась конвертированная цена
  ...
});

// Стало:
await addSubscription({
  price: selectedPlan.price, // ✅ Сохраняется цена в рублях как есть
  ...
});
```

### 3. `/src/app/components/AddSubscriptionModal.tsx`
**Добавлена конвертация в рубли при ручном вводе:**
```typescript
const priceInUserCurrency = parseFloat(formData.price);
const priceInRub = convertToRub(priceInUserCurrency, settings.currency);

await addSubscription({
  ...formData,
  price: priceInRub // ✅ Конвертируем в рубли перед сохранением
});
```

### 4. `/src/app/components/EditSubscriptionModal.tsx`
**Добавлена двусторонняя конвертация:**
```typescript
// При загрузке - конвертируем из RUB в валюту пользователя для отображения
const displayPrice = convertPrice(subscription.price);

// При сохранении - конвертируем обратно в RUB
const priceInUserCurrency = parseFloat(formData.price);
const priceInRub = convertToRub(priceInUserCurrency, settings.currency);

await updateSubscription(subscription.id, {
  price: priceInRub // ✅ Сохраняем в рублях
});
```

## Логика работы

### Для сервисов из базы (созданных владельцами)
1. Владельцы создают сервисы с ценами в рублях
2. Цены хранятся в таблице `services` в рублях
3. При выборе сервиса цена сохраняется в `subscriptions` **без конвертации** (в рублях)
4. При отображении цена конвертируется в валюту пользователя

### Для ручного добавления подписок
1. Пользователь вводит цену в **своей валюте** (USD, EUR и т.д.)
2. Цена конвертируется в рубли функцией `convertToRub`
3. В БД сохраняется цена в рублях
4. При отображении цена конвертируется обратно в валюту пользователя

### При редактировании подписки
1. Цена загружается из БД (в рублях)
2. Конвертируется в валюту пользователя для отображения
3. При сохранении конвертируется обратно в рубли

## Примеры

### Пример 1: Telegram Premium (300 RUB)
- Пользователь с валютой USD выбирает Telegram Premium
- В БД сохраняется: `300 RUB`
- При отображении: `300 / 80 = 3.75 USD` ✅

### Пример 2: Ручное добавление Netflix (15 USD)
- Пользователь с валютой USD вводит: `15`
- Конвертируется в рубли: `15 * 80 = 1200 RUB`
- В БД сохраняется: `1200 RUB`
- При отображении: `1200 / 80 = 15 USD` ✅

### Пример 3: Переключение валюты
- В БД хранится: `300 RUB`
- Пользователь переключает валюту с USD на EUR
- Отображается: `300 / 95 = 3.16 EUR` ✅
- Данные в БД не меняются, только отображение

## Проверка
Чтобы убедиться, что исправление работает:

1. Выберите подписку из каталога (например, Telegram Premium за 300₽)
2. Добавьте её в свои подписки
3. Перейдите в настройки и смените валюту на доллары (USD)
4. Проверьте, что цена отображается как `3.75 $` (а не `0.05 $`)
5. Смените валюту на евро (EUR)
6. Проверьте, что цена отображается как `3.16 €`
