## 🔒 Безопасность

### Row Level Security (RLS)

Все операции защищены на уровне базы данных:

```sql
-- Просмотр: ВСЕ пользователи
SELECT * FROM services;  ✅

-- Создание: ТОЛЬКО владельцы
INSERT INTO services;    ✅ (только max.sokolvp / belovodvadim)

-- Редактирование: ТОЛЬКО владельцы  
UPDATE services;         ✅ (только max.sokolvp / belovodvadim)

-- Удаление: ТОЛЬКО владельцы
DELETE FROM services;    ✅ (только max.sokolvp / belovodvadim)
```

### Каскадное удаление

При удалении сервиса **автоматически удаляются все подписки пользователей**:

```sql
-- Триггер: cascade_delete_service_subscriptions
DELETE FROM services WHERE id = 'xxx';
→ Автоматически: DELETE FROM subscriptions WHERE name = 'ServiceName';
```

### Realtime синхронизация

Все изменения синхронизируются в реальном времени через Supabase Realtime:
- ✅ Создание сервиса → видят все пользователи
- ✅ Редактирование → обновляется у всех
- ✅ Удаление → исчезает у всех + удаляются ��одписки

## 📊 Схема работы

### Владелец создает сервис

```
Владелец                          Supabase                    Все пользователи
   │                                  │                              │
   │──── Создает "Netflix" ────────>  │                              │
   │                                  │──── Realtime update ────────>│
   │                                  │                              │
   │<──── Сервис сохранен ────────────│                              │
   │                                  │                         "Netflix" доступен
```

### Пользователь добавляет подписку

```
Пользователь                     Supabase                    Владелец
   │                                  │                              │
   │──── Выбирает "Netflix" ──────>  │                              │
   │──── Выбирает тариф ───────────>  │                              │
   │──── Указывает дату ───────────>  │                              │
   │                                  │                              │
   │<──── Подписка создана ───────────│                              │
   │                                  │                    (не влияет на владельца)
```

### Владелец удаляет сервис

```
Владелец                          Supabase                    Пользователи
   │                                  │                              │
   │──── Удаляет "Netflix" ────────>  │                              │
   │                                  │──── Trigger CASCADE ─────────>│
   │                                  │     (удаляет подписки)        │
   │                                  │──── Realtime update ─────────>│
   │<──── Сервис удален ──────────────│                              │
   │                                  │                    Подписки удалены
```
