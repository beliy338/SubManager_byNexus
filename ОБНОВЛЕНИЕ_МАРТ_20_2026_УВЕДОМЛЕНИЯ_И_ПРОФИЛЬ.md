# Обновление от 20 марта 2026 - Уведомления и профиль

## Выполненные исправления

### 1. ✅ Исправлен значок количества уведомлений

**Проблема:** Значок с количеством непрочитанных уведомлений показывался только после открытия панели уведомлений.

**Решение:**
- Добавлен отдельный `useEffect` для загрузки количества уведомлений при изменении `user?.id`
- Добавлена периодическая проверка новых уведомлений каждые 30 секунд
- Улучшена обработка real-time обновлений из Supabase
- Теперь счетчик обновляется автоматически при:
  - Монтировании компонента
  - Изменении пользователя
  - Получении события из AppContext (через window events)
  - Получении real-time обновления из Supabase
  - Периодической проверке (каждые 30 секунд)

**Файл:** `/src/app/components/NotificationsButton.tsx`

## Технические детали

### Механизм отображения уведомлений

1. **Загрузка при монтировании:**
   ```typescript
   useEffect(() => {
     if (!user?.id) {
       setUnreadCount(0);
       return;
     }
     loadUnreadCount();
   }, [user?.id]);
   ```

2. **Периодическая проверка:**
   - Интервал: 30 секунд
   - Автоматически загружает новые уведомления

3. **Real-time обновления:**
   - Подписка на изменения таблицы `notifications` в Supabase
   - Обработка событий INSERT, UPDATE, DELETE

### Формат уведомления об изменении профиля

```typescript
{
  type: 'profile_updated',
  title: 'notification_profile_updated_title',
  message: 'notification_profile_updated_message',
  metadata: {
    changes: "Имя изменено с \"Старое\" на \"Новое\"; Номер телефона изменен с \"+71234567890\" на \"+79876543210\""
  }
}
```

## Известные ограничения

- Email нельзя изменить (это ограничение Supabase Auth), поэтому уведомления об изменении email не создаются
- Страна не отслеживается в уведомлениях (можно добавить при необходимости)
- Тестовые пользователи (test-token) не получают уведомления
