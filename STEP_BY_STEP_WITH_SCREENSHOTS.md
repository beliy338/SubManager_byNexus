# 📸 Пошаговая инструкция с описанием (без команд)

## ✅ ШАГ 1: Регистрация на Cloudflare

### Что делать:
1. Откройте браузер
2. Перейдите на: **https://dash.cloudflare.com/sign-up**
3. Заполните форму регистрации:
   - Email
   - Пароль
4. Нажмите **Sign Up**
5. Откройте почту и подтвердите email

**Время: 2 минуты**

---

## ✅ ШАГ 2: Создание Worker

### Что делать:
1. После входа на https://dash.cloudflare.com/ вы увидите главную панель
2. В **левом меню** найдите и нажмите **Workers & Pages**
3. Нажмите синюю кнопку **Create application**
4. Выберите вкладку **Workers** (если не выбрана)
5. Нажмите **Create Worker**
6. В поле "Name" введите: **submanager-supabase-proxy**
   (или любое другое имя без пробелов)
7. Нажмите **Deploy**

**Что вы увидите:**
- Worker будет создан с шаблонным кодом
- Откроется страница с вашим Worker

**Время: 1 минута**

---

## ✅ ШАГ 3: Замена кода Worker

### Что делать:
1. На странице Worker нажмите **Edit code** (или **Quick Edit**)
2. Откроется редактор кода с 3 панелями:
   - Слева: файловая структура
   - Центр: редактор кода
   - Справа: предпросмотр
3. В центральной панели **ВЫДЕЛИТЕ ВЕСЬ КОД** (Ctrl+A)
4. **УДАЛИТЕ** его (Delete)
5. Откройте файл **cloudflare-worker-code.js** из проекта
6. **СКОПИРУЙТЕ** весь код из этого файла (Ctrl+A, Ctrl+C)
7. **ВСТАВЬТЕ** в редактор Cloudflare (Ctrl+V)
8. Нажмите **Save and Deploy** (синяя кнопка справа вверху)

**Что вы увидите:**
- "Successfully deployed" (успешно развернуто)
- В правой панели может появиться тестовый результат

**Время: 2 минуты**

---

## ✅ ШАГ 4: Получение URL Worker

### Что делать:
1. Вернитесь на страницу Worker (если ещё в редакторе, нажмите стрелку назад)
2. На главной странице Worker найдите **"Preview"** или **"Triggers"**
3. Скопируйте URL вашего Worker

**Он будет выглядеть так:**
```
https://submanager-supabase-proxy.ваш-поддомен.workers.dev
```

**Или найдите в разделе:**
- Workers & Pages → Ваш Worker → Settings → Triggers → Routes

**ВАЖНО:** Скопируйте этот URL и сохраните где-нибудь!

**Время: 30 секунд**

---

## ✅ ШАГ 5: Проверка работы Worker

### Что делать:
1. Откройте новую вкладку браузера
2. Вставьте ваш URL Worker и добавьте `/rest/v1/`:
   ```
   https://submanager-supabase-proxy.ваш-поддомен.workers.dev/rest/v1/
   ```
3. Нажмите Enter

**Что вы должны увидеть:**
- JSON ответ с информацией о Supabase API
- Что-то вроде: `{"message":"The server is running..."}`
- Или список доступных таблиц

**Если видите ошибку:**
- Проверьте, правильно ли скопирован код
- Убедитесь, что URL правильный

**Время: 30 секунд**

---

## ✅ ШАГ 6: Настройка вашего приложения

### Вариант А: Если у вас есть доступ к файлам проекта

1. Откройте папку вашего проекта SubManager
2. Создайте новый файл `.env.production` (если его нет)
3. Добавьте в него строку:
   ```env
   VITE_SUPABASE_PROXY_URL=https://submanager-supabase-proxy.ваш-поддомен.workers.dev
   ```
   (замените на ваш реальный URL)
4. Сохраните файл

### Вариант Б: Если используете хостинг с переменными окружения

Если вы деплоите на:
- **Vercel**: Settings → Environment Variables
- **Netlify**: Site settings → Environment variables
- **GitHub Pages**: Actions → Secrets

Добавьте переменную:
- **Имя:** `VITE_SUPABASE_PROXY_URL`
- **Значение:** `https://submanager-supabase-proxy.ваш-поддомен.workers.dev`

**Время: 1 минута**

## ✅ ШАГ 7: Пересборка приложения

### Если у вас ЕСТЬ доступ к командной строке:
```bash
npm run build
```

### Если у вас НЕТ доступа к командной строке:

**Для Vercel:**
1. Откройте ваш проект на vercel.com
2. Нажмите **Deployments**
3. Нажмите **Redeploy** на последнем деплое

**Для Netlify:**
1. Откройте ваш сайт на netlify.com
2. Нажмите **Deploys**
3. Нажмите **Trigger deploy** → **Deploy site**

**Для GitHub Pages:**
1. Откройте репозиторий на GitHub
2. Перейдите в **Actions**
3. Выберите последний workflow
4. Нажмите **Re-run all jobs**

**Время: зависит от платформы (обычно 2-5 минут)**

## ❓ Часто задаваемые вопросы

**Q: Можно ли использовать свой домен вместо workers.dev?**
A: Да! В настройках Worker → Triggers → Custom Domains → Add Custom Domain

**Q: Это бесплатно?**
A: Да, до 100,000 запросов в день

**Q: Как обновить код Worker?**
A: Просто откройте Worker → Edit code → внесите изменения → Save and Deploy

**Q: Worker не работает, что делать?**
A:
1. Проверьте логи: Workers & Pages → Ваш Worker → Logs → Begin log stream
2. Убедитесь, что код скопирован полностью
3. Проверьте статус Cloudflare: www.cloudflarestatus.com
