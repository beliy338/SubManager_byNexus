# Добавление статуса прочтения сообщений в систему поддержки

## Обзор изменений

Этот апдейт добавляет функционал отслеживания прочитанных/непрочитанных сообщений в систему поддержки SubManager. Теперь:

1. **Для пользователей**: Видны галочки в их собственных сообщениях (1 галочка - непрочитано, 2 галочки - прочитано)
2. **Для владельцев**: 
   - Видны галочки в их ответах (1 галочка - непрочитано, 2 галочки - прочитано)
   - Счётчик непрочитанных сообщений около каждого пользователя в списке
   - Общий счётчик непрочитанных на кнопке поддержки
3. **Автоматическое**: Сообщения отмечаются как прочитанные когда пользователь открывает чат

## Инструкция по применению

### Шаг 1: Выполните SQL скрипт

1. Откройте Supabase Dashboard
2. Перейдите в раздел **SQL Editor**
3. Создайте новый запрос
4. Скопируйте содержимое файла `add-is-read-column.sql` и вставьте в редактор
5. Нажмите **Run** для выполнения скрипта

### Что делает скрипт:

- Добавляет колонку `is_read` (boolean, default: false) в таблицу `support_messages`
- Создаёт индексы для оптимизации запросов по статусу прочтения
- Добавляет RLS политики для обновления статуса прочтения:
  - Пользователи могут отмечать как прочитанные ответы поддержки на их сообщения
  - Владельцы могут отмечать как прочитанные сообщения пользователей

### Шаг 2: Проверка выполнения

После выполнения SQL скрипта, выполните эту проверочную команду в SQL Editor:

```sql
SELECT column_name, data_type, column_default 
FROM information_schema.columns 
WHERE table_name = 'support_messages' AND column_name = 'is_read';
```

Ожидаемый результат:
```
column_name | data_type | column_default
is_read     | boolean   | false
```

### Шаг 3: Проверка политик

Проверьте, что политики созданы корректно:

```sql
SELECT policyname, permissive, roles, cmd 
FROM pg_policies 
WHERE tablename = 'support_messages' 
AND policyname LIKE '%read%';
```

Должны быть видны две новые политики:
- `Users can mark admin replies as read`
- `Owners can mark user messages as read`

## Функционал в интерфейсе

### Для обычных пользователей:

1. **В чате**: 
   - Собственные сообщения показывают галочку статуса справа внизу
   - ✓ (одна галочка) = непрочитано владельцами
   - ✓✓ (две галочки) = прочитано владельцами
   - Градиент сообщений: сине-фиолетовый

2. **Счётчик непрочитанных**:
   - Красный бейдж на кнопке поддержки показывает количество непрочитанных ответов от поддержки
   - Автоматически обновляется в реальном времени
   - Исчезает когда все сообщения прочитаны

3. **Автоматическая отметка**:
   - Когда пользователь открывает чат, все непрочитанные ответы от поддержки автоматически отмечаются как прочитанные
   - Счётчик обнуляется

### Для владельцев (max.sokolvp@gmail.com, belovodvadim@gmail.com):

1. **Список пользователей**:
   - Около каждого пользователя показывается "Непрочитанных: X" если есть непрочитанные сообщения
   - Счётчик исчезает когда владелец открывает чат с пользователем

2. **В чате**:
   - Собственные ответы (admin replies) показывают галочку статуса
   - ✓ (одна галочка) = непрочитано пользователем
   - ✓✓ (две галочки) = прочитано пользователем
   - Градиент ответов: сине-фиолетовый

3. **Счётчик непрочитанных**:
   - Красный бейдж на кнопке поддержки показывает общее количество непрочитанных сообщений от всех пользователей
   - Автоматически обновляется в реальном времени

4. **Автоматическая отметка**:
   - Когда владелец открывает чат с конкретным пользователем, все непрочитанные сообщения от этого пользователя автоматически отмечаются как прочитанные
   - Счётчик около этого пользователя обнуляется

## Структура данных

### Таблица support_messages

```sql
CREATE TABLE support_messages (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id),
  user_email TEXT,
  user_name TEXT,
  message TEXT,
  created_at TIMESTAMP WITH TIME ZONE,
  is_admin_reply BOOLEAN,
  parent_message_id UUID REFERENCES support_messages(id),
  is_read BOOLEAN DEFAULT false  -- НОВОЕ ПОЛЕ
);
```

## Логика подсчёта непрочитанных

### Для пользователей:
Считаются непрочитанные ответы от поддержки (admin replies) на их сообщения:
```typescript
unreadCount = messages.filter(msg => 
  msg.is_admin_reply && 
  msg.parent_message_id in userMessageIds &&
  !msg.is_read
).length
```

### Для владельцев:
Считаются все непрочитанные сообщения от пользователей (не admin replies):
```typescript
unreadCount = messages.filter(msg => 
  !msg.is_admin_reply && 
  !msg.is_read
).length
```
