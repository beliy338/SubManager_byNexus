# 🚀 Применение исправления рекурсии - Пошаговая инструкция

## Проблема
При загрузке чата поддержки возникает ошибка:
```json
{
  "code": "42P17",
  "message": "infinite recursion detected in policy for relation \"support_messages\""
}
```

---

## ✅ Решение - Выполните SQL скрипт в Supabase

### Шаг 1: Войдите в Supabase Dashboard
1. Откройте браузер
2. Перейдите на **https://supabase.com**
3. Войдите в свой аккаунт
4. Выберите проект **SubManager**

### Шаг 2: Откройте SQL Editor
1. В левом меню найдите **SQL Editor**
2. Нажмите на него

### Шаг 3: Создайте новый запрос
1. Нажмите кнопку **New query** (или **+ New query**)
2. Откроется пустой редактор SQL

### Шаг 4: Скопируйте SQL скрипт
1. Откройте файл **`fix-support-messages-recursion.sql`** в вашем проекте
2. Выделите ВСЁ содержимое файла (Ctrl+A)
3. Скопируйте (Ctrl+C)

### Шаг 5: Вставьте и выполните
1. Вставьте скопированный код в SQL Editor (Ctrl+V)
2. Нажмите кнопку **Run** внизу справа
   - Или используйте горячую клавишу **Ctrl+Enter** (Windows/Linux)
   - Или **Cmd+Enter** (Mac)

### Шаг 6: Проверьте результат
Вы должны увидеть в результатах выполнения:
```
NOTICE: Support messages RLS policies have been fixed!
NOTICE: Users can now view:
NOTICE:   1. Their own messages
NOTICE:   2. Admin replies to their messages
NOTICE:   3. Owners can see all messages
```

### Шаг 7: Обновите приложение
1. Вернитесь в приложение SubManager
2. Обновите страницу (F5 или Ctrl+R)
3. Откройте чат поддержки (кнопка в правом нижнем углу)
4. Чат должен загрузиться без ошибок! ✅

## 🔍 Что было исправлено

### До исправления:
```sql
-- ❌ Эта политика вызывала бесконечную рекурсию
parent_message_id IN (
    SELECT id FROM support_messages WHERE user_id = auth.uid()
)
```

### После исправления:
```sql
-- ✅ Используется функция-помощник без рекурсии
public.is_user_parent_message(parent_message_id, auth.uid())
```

## 🆘 Если что-то пошло не так

### Ошибка при выполнении SQL
- Убедитесь, что скопировали весь скрипт целиком
- Проверьте, что вы вошли под правильным аккаунтом в Supabase
- Попробуйте выполнить скрипт снова

### Ошибка всё ещё появляется
1. Очистите кэш браузера (Ctrl+Shift+Delete)
2. Перезагрузите страницу приложения
3. Проверьте в Supabase Dashboard → Authentication → Policies, что политика создана

### Чат не загружается
1. Откройте консоль браузера (F12)
2. Посмотрите на ошибки в разделе Console
3. Если видите другую ошибку - сообщите владельцу
