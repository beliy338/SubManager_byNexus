# Исправление ошибок базы данных

## Проблемы
1. Ошибка "permission denied for table users" при загрузке сообщений поддержки
2. Аккаунт `belovodvadim@gmail.com` не может добавлять новые сервисы

## Решение

### Шаг 1: Создайте таблицу support_messages

1. Откройте **Supabase Dashboard** → выберите ваш проект
2. Перейдите в **SQL Editor** (боковая панель слева)
3. Создайте новый запрос и выполните содержимое файла **`supabase-support-messages-table.sql`**

Этот скрипт создаст:
- Таблицу `support_messages` для чата поддержки
- RLS политики для безопасного доступа
- Индексы для производительности

### Шаг 2: Обновите RLS политики для таблицы services

1. В том же **SQL Editor** выполните содержимое файла **`fix-owner-policies.sql`**

Этот скрипт:
- Удалит старые политики для таблицы `services`
- Создаст новые политики с проверкой email без учета регистра
- Проверит, что политики созданы правильно

### Шаг 3: Проверьте email в Supabase Auth

1. В Supabase Dashboard перейдите в **Authentication** → **Users**
2. Найдите пользователя с email `belovodvadim@gmail.com`
3. Убедитесь, что email записан именно как `belovodvadim@gmail.com` (в нижнем регистре)

### Шаг 4: Проверьте работу

1. **Перезагрузите приложение** (обновите страницу в браузере)
2. Проверьте, что:
   - Чат поддержки открывается без ошибок
   - Владелец `belovodvadim@gmail.com` может добавлять сервисы
   - Обычные пользователи могут отправлять сообщения в поддержку

## Структура таблицы support_messages

```sql
id                  UUID (PRIMARY KEY)
user_id             UUID (REFERENCES auth.users)
user_email          TEXT
user_name           TEXT
message             TEXT
is_admin_reply      BOOLEAN
parent_message_id   UUID (REFERENCES support_messages)
created_at          TIMESTAMP
```

## Права доступа

### Таблица support_messages:
- **Обычные пользователи:**
  - Могут видеть только свои сообщения
  - Могут отправлять сообщения
  - Могут редактировать/удалять свои сообщения в течение 5 минут

- **Владельцы (max.sokolvp@gmail.com, belovodvadim@gmail.com):**
  - Видят все сообщения всех пользователей
  - Могут отправлять ответы (is_admin_reply = true)
  - Могут редактировать/удалять любые сообщения

### Таблица services:
- **Только владельцы** могут создавать, редактировать и удалять сервисы
- **Все пользователи** могут просматривать сервисы

## Диагностика проблем

### Если чат поддержки не работает:

1. Откройте консоль браузера (F12)
2. Попробуйте открыть чат поддержки
3. Проверьте ошибки:
   - Если "permission denied" - таблица не создана или RLS политики неверны
   - Если "relation does not exist" - таблица `support_messages` не создана

### Если владелец не может добавлять сервисы:

1. Откройте консоль браузера (F12)
2. Попробуйте добавить сервис
3. Проверьте логи:
   ```
   Creating service with user: belovodvadim@gmail.com
   isOwner check: { email: "belovodvadim@gmail.com", isOwner: true }
   ```
4. Если `isOwner: false` - email в базе отличается
5. Если код ошибки `42501` - RLS политики не обновлены

## Файлы для выполнения в Supabase

Выполните в следующем порядке:

1. **`supabase-support-messages-table.sql`** - создание таблицы поддержки
2. **`fix-owner-policies.sql`** - исправление политик для сервисов

## Важно

⚠️ После выполнения SQL скриптов обязательно **перезагрузите приложение** в браузере, чтобы изменения вступили в силу.
