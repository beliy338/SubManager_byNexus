# Getting Started with SubManager

## 🎉 Your SubManager is Ready!

SubManager теперь полностью использует облачное хранилище данных на базе Supabase. Все данные пользователей надежно хранятся в облаке с полной изоляцией между аккаунтами.

## ✅ Что было сделано

### 1. Облачное хранилище
- ❌ Удалена вся логика localStorage
- ❌ Удалены тестовые аккаунты
- ✅ Все данные теперь хранятся в Supabase
- ✅ Каждый пользователь имеет изолированный аккаунт
- ✅ Автоматическая синхронизация между устройствами

### 2. Аутентификация
- ✅ Полная интеграция с Supabase Auth
- ✅ Регистрация через email/пароль
- ✅ Безопасный вход в систему
- ✅ Автоматическое подтверждение email (для разработки)

### 3. Сохранение данных
Все данные каждого пользователя сохраняются в облаке:
- ✅ Подписки (добавление, редактирование, удаление)
- ✅ Настройки (язык, валюта, размер шрифта, тема)
- ✅ Персональные предпочтения

### 4. Безопасность
- ✅ Токены доступа для всех API запросов
- ✅ Проверка авторизации на сервере
- ✅ Изоляция данных между пользователями
- ✅ Безопасные HTTPS соединения

## 🚀 Как начать пользоваться

### Для новых пользователей:

1. **Откройте приложение**
   - Вы увидите страницу входа

2. **Создайте аккаунт**
   - Нажмите "Sign Up"
   - Введите имя, email и пароль (минимум 6 символов)
   - Нажмите "Create Account"

3. **Автоматический вход**
   - После регистрации вы автоматически войдете в систему
   - Увидите приветственное сообщение с инструкциями

4. **Начните добавлять подписки**
   - Нажмите кнопку "+ Add Subscription"
   - Заполните данные подписки
   - Всё автоматически сохранится в облаке!

### Для существующих пользователей:

1. **Войдите в систему**
   - Используйте свой email и пароль
   - Все ваши данные загрузятся автоматически

2. **Ваши данные синхронизированы**
   - Все подписки на месте
   - Настройки сохранены
   - Можете войти с любого устройства

## 🔧 Техническая информация

### Архитектура
```
Frontend (React) 
    ↓ 
Supabase Auth (Аутентификация)
    ↓
Edge Functions (API сервер)
    ↓
KV Store (База данных)
```

### Эндпоинты API
- `POST /signup` - Регистрация
- `GET /subscriptions` - Получение подписок
- `POST /subscriptions` - Добавление подписки
- `PUT /subscriptions/:id` - Обновление подписки
- `DELETE /subscriptions/:id` - Удаление подписки
- `GET /settings` - Получение настроек
- `PUT /settings` - Обновление настроек

### Структура хранения данных
```
subscriptions:{userId}:{subscriptionId} → Данные подписки
settings:{userId} → Настройки пользователя
```

## ⚠️ Важно знать

1. **Пароли**
   - Минимум 6 символов
   - Безопасно хранятся в зашифрованном виде
   - Сброс пароля пока не реализован

2. **Email**
   - Используется только для входа
   - Автоматически подтверждается (режим разработки)
   - Не используется для рассылок

3. **Данные**
   - Полностью изолированы между пользователями
   - Хранятся в облаке Supabase
   - Автоматически резервируются
