## 🎯 Цель

Исправить ошибку бесконечной рекурсии в Row Level Security (RLS) политиках таблицы `support_messages`, которая приводила к невозможности загрузки чата поддержки.

---

## ❌ Проблема

### Симптомы
- При открытии чата поддержки появляется ошибка
- В консоли браузера: `infinite recursion detected in policy for relation "support_messages"`
- Код ошибки PostgreSQL: `42P17`
- Сообщения не загружаются
- Невозможно отправить новые сообщения

### Причина
Политика RLS для таблицы `support_messages` содержала подзапрос к той же таблице:

```sql
parent_message_id IN (
    SELECT id FROM support_messages WHERE user_id = auth.uid()
)
```

При применении политики PostgreSQL пытался применить ту же политику к подзапросу, что создавало бесконечную рекурсию.

---

## ✅ Решение

### Подход
Создана функция-помощник `is_user_parent_message()` с атрибутом `SECURITY DEFINER`, которая:
- Обходит RLS (работает с правами создателя функции)
- Безопасно проверяет принадлежность сообщений пользователю
- Не создаёт рекурсии

### Архитектура решения
1. **Функция-помощник** с `SECURITY DEFINER`
2. **Обновлённая RLS политика** использующая функцию вместо подзапроса
3. **Права доступа** для authenticated пользователей

## 🔧 Техническое решение

### Было (❌ рекурсия)

```sql
CREATE POLICY "Users can view messages"
    ON support_messages
    FOR SELECT
    USING (
        user_id = auth.uid()
        OR parent_message_id IN (
            SELECT id FROM support_messages -- ← Рекурсия!
            WHERE user_id = auth.uid()
        )
    );
```

### Стало (✅ без рекурсии)

**Шаг 1: Функция**
```sql
CREATE FUNCTION is_user_parent_message(parent_id UUID, user_uuid UUID)
RETURNS BOOLEAN
LANGUAGE plpgsql
SECURITY DEFINER  -- Обходит RLS
SET search_path = public  -- Безопасность
AS $$
BEGIN
    RETURN EXISTS (
        SELECT 1 FROM support_messages 
        WHERE id = parent_id AND user_id = user_uuid
    );
END;
$$;
```

**Шаг 2: Политика**
```sql
CREATE POLICY "Users can view messages"
    ON support_messages
    FOR SELECT
    USING (
        user_id = auth.uid()
        OR (
            is_admin_reply = true
            AND is_user_parent_message(parent_message_id, auth.uid())
            -- ✅ Функция вместо подзапроса!
        )
    );
```
