## 📊 Техническая информация

### Причина ошибки
RLS политика делала подзапрос к той же таблице, создавая бесконечную рекурсию.

### Решение
Создана функция `is_user_parent_message()` с `SECURITY DEFINER`, которая обходит RLS.

### Безопасность
- ✅ `SECURITY DEFINER` используется безопасно
- ✅ `SET search_path = public` для защиты
- ✅ RLS активен для всех операций
- ✅ Права доступа корректны

## 🎓 Для дальнейшего изучения

### Что такое RLS?
Row Level Security - система безопасности PostgreSQL на уровне строк.

### Что такое SECURITY DEFINER?
Атрибут функции, позволяющий ей работать с правами создателя, обходя RLS.

### Почему возникла рекурсия?
Политика RLS делала подзапрос к той же таблице, на которую она применялась.

### Как избежать в будущем?
Использовать функции с `SECURITY DEFINER` вместо подзапросов в RLS политиках.
