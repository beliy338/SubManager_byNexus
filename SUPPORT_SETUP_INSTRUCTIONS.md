# 🚀 Инструкция по настройке системы поддержки SubManager

## 📋 Описание

Этот документ содержит инструкции по единоразовой настройке системы поддержки в Supabase для приложения SubManager.

## ✅ Что было реализовано

1. **Исправлены опечатки** в админской версии ("Упрвление сервисами" → "Управление сервисами")
2. **Добавлена прокрутка** в чате поддержки для обеих версий (пользователи и владельцы)
3. **Единый SQL скрипт** для автоматической настройки таблицы поддержки
4. **Календарь платежей** в аналитике с:
   - Переключением месяцев и годов
   - Выделением дней со списаниями сине-фиолетовым градиентом
   - Модальным окном с информацией о подписках при нажатии на день

## 🔧 Установка системы поддержки (ОДИН РАЗ)

### Шаг 1: Откройте Supabase Dashboard

1. Перейдите на [Supabase Dashboard](https://supabase.com/dashboard)
2. Выберите ваш проект
3. В левом меню выберите **SQL Editor**

### Шаг 2: Выполните SQL скрипт

1. Нажмите **"New Query"**
2. Откройте файл `/supabase-support-complete-setup.sql`
3. Скопируйте **весь** код из файла
4. Вставьте в SQL Editor
5. Нажмите **"Run"** (или F5)

### Шаг 3: Проверьте выполнение

После выполнения скрипта вы должны увидеть сообщение:
```
Support messages table setup completed successfully!
```

### Шаг 4: Проверьте таблицу

1. В левом меню выберите **Table Editor**
2. Найдите таблицу `support_messages`
3. Убедитесь что она содержит следующие поля:
   - `id` (UUID)
   - `user_id` (UUID)
   - `user_email` (TEXT)
   - `user_name` (TEXT)
   - `message` (TEXT)
   - `is_admin_reply` (BOOLEAN)
   - `parent_message_id` (UUID)
   - `is_read` (BOOLEAN)
   - `created_at` (TIMESTAMP)

### Функционал:

1. **Переключение месяцев/годов** с помощью стрелок
2. **Кнопка "Сегодня"** для быстрого возврата
3. **Выделение дней** со списаниями градиентом от синего к фиолетовому

### Расположение:

Календарь находится в разделе **"Аналитика"** под графиком динамики расходов.

## 🐛 Устранение неполадок

### Если сообщения не отображаются:

1. Проверьте что вы выполнили SQL скрипт
2. Проверьте консоль браузера (F12) на наличие ошибок
3. Убедитесь что вы авторизованы
4. Попробуйте обновить страницу

### Если ошибка "table does not exist":

Выполните SQL скрипт `/supabase-support-complete-setup.sql` в Supabase Dashboard.

### Если ошибка прав доступа:

Проверьте что RLS политики были созданы корректно. Повторно выполните SQL скрипт.

## 📝 Технические детали

### Файлы системы:

- `/supabase-support-complete-setup.sql` - Единый SQL скрипт установки
- `/src/app/components/SupportModal.tsx` - Компонент модального окна чата
- `/src/app/components/SupportButton.tsx` - Кнопка вызова поддержки
- `/src/app/components/PaymentCalendar.tsx` - Компонент календаря платежей

### RLS политики:

1. **SELECT**: Пользователи видят свои сообщения + ответы на них, владельцы видят всё
2. **INSERT**: Все могут создавать сообщения
3. **UPDATE**: Пользователи обновляют свои недавние сообщения, владельцы обновляют любые
4. **DELETE**: Пользователи удаляют свои недавние сообщения, владельцы удаляют любые
