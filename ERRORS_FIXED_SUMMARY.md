## 2. 🚨 Infinite Recursion in support_messages RLS Policy

### Проблема:
```
Error: infinite recursion detected in policy for relation "support_messages"
Code: 42P17
```

Политики Row Level Security (RLS) для таблицы `support_messages` вызывали бесконечную рекурсию при попытке проверить, может ли пользователь видеть ответы администратора.

### Причина:
Политика SELECT для `support_messages` пыталась выполнить подзапрос к той же таблице внутри условия `USING`, что создавало циклическую зависимость.

### Исправление:
✅ **Уже существует готовое решение** в файле `/fix-support-messages-recursion.sql`

Этот скрипт:
1. Создаёт функцию `is_user_parent_message()` с флагом `SECURITY DEFINER`, которая проверяет принадлежность сообщения **БЕЗ вызова RLS политик**
2. Удаляет проблемную политику
3. Создаёт новую исправленную политику, использующую эту функцию

### Как применить:
1. Откройте Supabase Dashboard → SQL Editor
2. Скопируйте весь код из `/fix-support-messages-recursion.sql`
3. Вставьте и запустите (Run)
4. Перезагрузите приложение

### Документация:
📄 Подробная инструкция: `/FIX_SUPPORT_RECURSION_NOW.md`

### Результат после применения:
✅ Чат поддержки работает без ошибок  
✅ Пользователи видят свои сообщения и ответы администратора  
✅ Владельцы (owners) видят все сообщения  
✅ Нет циклических зависимостей в RLS политиках
