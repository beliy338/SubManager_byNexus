# Обновление: Исправление системы логотипов и скроллбара поддержки

**Дата:** 16 марта 2026  
**Статус:** ✅ Завершено

## Проблемы
1. **Логотипы сервисов**: Отображались градиентные буквы вместо фотографий, загруженных админом
2. **Чат поддержки**: Отсутствовал видимый скроллбар, невозможно было прокручивать историю колёсиком мыши

## Внесённые изменения

### 1. Улучшена система логотипов (ServiceLogo)
**Файл:** `/src/app/components/ServiceLogo.tsx`

**Что изменилось:**
- ✅ Добавлено состояние `imageError` для отслеживания ошибок загрузки
- ✅ Добавлен обработчик `onError` для автоматического fallback на градиентную букву при ошибке загрузки
- ✅ Добавлен фоновый цвет `bg-muted` для лучшего отображения прозрачных изображений
- ✅ Компонент теперь умно определяет, когда показывать фото, а когда - первую букву

**Как работает:**
```tsx
// Если URL логотипа существует и не было ошибки - показываем фото
if (logo && !imageError) {
  return <img onError={() => setImageError(true)} />
}
// Иначе - показываем градиентную букву (fallback)
```

### Таблица `services` (создаётся владельцами):
- `icon` (text) - URL логотипа сервиса

### Таблица `subscriptions` (создаётся пользователями):
- `logo` (text) - URL логотипа, копируется из `services.icon` при создании

## Как это работает для пользователей

1. **Владелец создаёт сервис:**
   - Загружает логотип (base64 или URL)
   - Сохраняется в поле `services.icon`

2. **Пользователь выбирает сервис:**
   - ServiceLogo получает `service.icon`
   - При добавлении подписки логотип копируется в `subscriptions.logo`

3. **Отображение логотипов:**
   - Если URL валидный и картинка загружается → показывается фото
   - Если URL пустой или ошибка загрузки → показывается градиентная буква

4. **Чат поддержки:**
   - Видимый скроллбар с цветами темы
   - Прокр��тка колёсиком мыши работает автоматически
   - Адаптация под светлую/тёмную тему
