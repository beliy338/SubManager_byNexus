# 🎉 Обновление от 16 марта 2026 — Аналитика и Поддержка

## 📊 Что нового

### 1. Улучшенная аналитика затрат ✨

#### Новые возможности визуализации

**Три вкладки для разных представлений:**

##### 📈 Обзор
- **Карточки-метрики** с общей статистикой:
  - Всего расходов в месяц
  - Всего расходов в год
  - Количество активных подписок
- **Круговая диаграмма** — расходы по категориям с процентами
- **Столбчатая диаграмма** — топ-5 самых дорогих подписок
- **Линейный график** с выбором периода:
  - По месяцам (6 месяцев)
  - По кварталам (4 квартала)
  - По полугодиям (2 полугодия)

##### 🔍 Детализация
- **Таблица всех сервисов** с полной информацией:
  - Название сервиса
  - Категория
  - Цена в месяц (приведенная)
  - Цикл оплаты
  - Полная цена
  - Итоговая строка с суммами
- **Прогресс-бары по категориям**:
  - Визуальное представление доли каждой категории
  - Сумма и процент от общих расходов
  - Цветовая кодировка

##### 📅 Календарь
- **Компактный календарь** платежей
- Показывает дни с предстоящими списаниями
- Клик по дню показывает детали платежей

#### Технические улучшения

```typescript
// Группировка по периодам

- month: 6 месяцев назад

- year: 2 полугодия назад (по 6 месяцев)
```

**Оптимизация:**
- Все расчеты в реальном времени
- Автоматический пересчет при изменении валюты
- Умные группировки и сортировки

#### Возможности скрипта

**1. Умное создание/обновление таблицы**
```sql
-- Проверяет существование таблицы
CREATE TABLE IF NOT EXISTS support_messages (...)

-- Добавляет поля, если их нет
IF NOT EXISTS (SELECT ... WHERE column_name = 'is_read')
  ADD COLUMN is_read BOOLEAN DEFAULT false
```

**2. Helper функция против рекурсии**
```sql
CREATE OR REPLACE FUNCTION is_user_parent_message(...)
SECURITY DEFINER  -- Минует RLS, предотвращает рекурсию
```

**3. Четыре RLS политики**
- **SELECT**: Пользователи видят свои сообщения + админ-ответы
- **INSERT**: Пользователи создают свои, владельцы — любые
- **UPDATE**: 5 мин на редактирование + is_read для всех
- **DELETE**: 5 мин на удаление для пользователей

**4. Оптимизация производительности**
```sql
-- 6 индексов для быстрой работы:
support_messages_user_id_idx
support_messages_created_at_idx
support_messages_is_admin_reply_idx
support_messages_parent_id_idx
support_messages_is_read_idx
support_messages_user_is_read_idx (составной)
```

**5. Автоматическая проверка**
```sql
-- Выводит статистику после выполнения:
Колонок в таблице: 9
RLS политик создано: 4
Индексов создано: 6+
✓ ВСЁ НАСТРОЕНО КОРРЕКТНО!
```

#### Безопасность запуска

✅ **Можно запускать:**
- На пустой базе (создаст таблицу)
- На существующей таблице (обновит структуру)
- Повторно (безопасно, использует IF NOT EXISTS)

❌ **НЕ делает:**
- НЕ удаляет данные из таблицы
- НЕ требует предварительной очистки
- НЕ создает конфликтов

### SQL структура

```sql
TABLE support_messages
  id UUID PRIMARY KEY
  user_id UUID → auth.users(id)
  user_email TEXT
  user_name TEXT
  message TEXT
  is_admin_reply BOOLEAN
  parent_message_id UUID → support_messages(id)
  is_read BOOLEAN          -- ← Новое поле
  created_at TIMESTAMPTZ

FUNCTION is_user_parent_message(UUID, UUID) → BOOLEAN
  SECURITY DEFINER         -- ← Предотвращает рекурсию

POLICIES (4)
  SELECT, INSERT, UPDATE, DELETE

INDEXES (6)
  user_id, created_at, is_admin_reply,
  parent_id, is_read, (user_id + is_read)
```
