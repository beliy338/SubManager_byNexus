### 🔧 Технические детали

#### Структура базы данных:

```sql
CREATE TABLE public.services (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES auth.users(id),
    name TEXT NOT NULL,
    category TEXT NOT NULL,
    description TEXT,
    icon TEXT,
    is_popular BOOLEAN DEFAULT false,
    pricing_plans JSONB DEFAULT '[]'::jsonb,
    created_at TIMESTAMP WITH TIME ZONE,
    updated_at TIMESTAMP WITH TIME ZONE
);
```

#### Триггеры:
1. `update_services_updated_at_trigger` - обновляет `updated_at`
2. `cascade_delete_service_subscriptions` - удаляет подписки при удалении сервиса

#### Политики RLS:
1. `Anyone can view services` - чтение для всех
2. `Only owners can insert services` - создание только для владельцев
3. `Only owners can update services` - редактирование только для владельцев
4. `Only owners can delete services` - удаление только для владельцев

### 🚀 Как начать использовать

1. Выполните SQL-скрипт из `/supabase-services-table.sql` в Supabase
2. Войдите как владелец (max.sokolvp@gmail.com или belovodvadim@gmail.com)
3. Перейдите во вкладку "Подписки"
4. Создайте первые сервисы для пользователей
5. Обычные пользователи смогут выбирать из созданных сервисов

### ⚠️ Важные замечания

1. **Каскадное удаление необратимо** - при удалении сервиса удаляются все подписки пользователей
2. **Два владельца** - оба имеют одинаковые права
3. **Realtime обновления** - изменения синхронизируются в реальном времени
4. **Валютная конвертация** - цены автоматически конвертируются в валюту пользователя
