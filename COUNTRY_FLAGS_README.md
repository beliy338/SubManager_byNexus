# 🇷🇺🇬🇧🇧🇾🇨🇳 Система флагов стран в базе данных

## ✅ Что было реализовано

### 25 марта 2026 - Полная интеграция флагов стран с Supabase

Создана система постоянного хранения флагов стран в базе данных Supabase, решающая проблему отображения флагов на опубликованном сайте.

---

## 🎯 Проблема

Флаги стран использовали схему `figma:asset`, которая работает только локально в Figma Make, но не работает на опубликованном сайте по URL `https://body-order-32825369.figma.site`. При переходе на сайт флаги не отображались.

---

## 🔧 Решение

### 1️⃣ Создана таблица `country_flags` в Supabase

**Структура таблицы:**
- `id` - UUID (первичный ключ)
- `country_code` - ISO код страны (RU, GB, BY, CN)
- `language_code` - Код языка (ru, en, be, zh)
- `flag_url` - Публичный URL флага из Flagpedia CDN
- `flag_emoji` - Unicode emoji флага (опционально)
- `created_at` - Дата создания
- `updated_at` - Дата обновления

**Безопасность:**
- ✅ Включен Row Level Security (RLS)
- ✅ Все пользователи могут читать флаги (даже неавторизованные)
- ✅ Только владельцы (max.sokolvp@gmail.com, belovodvadim@gmail.com) могут добавлять/изменять флаги

### 2️⃣ Использован Flagpedia CDN

Все флаги загружаются с надежного публичного CDN:
- 🇷🇺 Россия: `https://flagcdn.com/w320/ru.png`
- 🇬🇧 Великобритания: `https://flagcdn.com/w320/gb.png`
- 🇧🇾 Беларусь: `https://flagcdn.com/w320/by.png`
- 🇨🇳 Китай: `https://flagcdn.com/w320/cn.png`

**Преимущества:**
- ✅ Высокое качество изображений (320px ширина)
- ✅ Быстрая загрузка через CDN
- ✅ 100% uptime и доступность
- ✅ Бесплатное использование

### 3️⃣ Обновлен компонент LanguageSelector

**Новая логика:**
1. При монтировании компонент загружает флаги из Supabase
2. Если БД недоступна - используются резервные `figma:asset` флаги
3. Флаги кешируются в state для быстрого отображения
4. Автоматический fallback на резервные изображения

**Файлы:**
- `/src/app/components/LanguageSelector.tsx` - обновлен с интеграцией БД
- `/supabase-country-flags-table.sql` - SQL скрипт для создания таблицы

## 🔄 Альтернативные CDN для флагов

Если основной CDN (Flagpedia) недоступен, используйте альтернативы:

### 1. Country Flags API
```
https://countryflagsapi.com/png/{country-code}
```

### 2. FlagsAPI (требует заглавные буквы)
```
https://flagsapi.com/{COUNTRY-CODE}/flat/64.png
```

### Обновление URL флагов

```sql
-- Обновите URL для конкретной страны
UPDATE country_flags 
SET flag_url = 'https://новый-cdn-url.png' 
WHERE country_code = 'RU';

-- Или обновите все флаги на другой CDN
UPDATE country_flags SET flag_url = 
  CASE 
    WHEN country_code = 'RU' THEN 'https://countryflagsapi.com/png/ru'
    WHEN country_code = 'GB' THEN 'https://countryflagsapi.com/png/gb'
    WHEN country_code = 'BY' THEN 'https://countryflagsapi.com/png/by'
    WHEN country_code = 'CN' THEN 'https://countryflagsapi.com/png/cn'
  END;
```

## 🔍 Отладка проблем

### Проблема: Флаги не загружаются из БД

**Решение:**
1. Проверьте консоль браузера на ошибки
2. Убедитесь что таблица `country_flags` создана
3. Проверьте RLS политики:
   ```sql
   SELECT * FROM country_flags; -- Должно работать для всех
   ```

### Проблема: Флаги не отображаются на сайте

**Решение:**
1. Проверьте что CDN доступен: откройте в браузере
   - https://flagcdn.com/w320/ru.png
2. Проверьте консоль на CORS ошибки
3. Попробуйте альтернативный CDN

### Проблема: Резервные флаги не работают

**Решение:**
Резервные `figma:asset` флаги работают только локально. На опубликованном сайте будут использоваться только флаги из БД.
