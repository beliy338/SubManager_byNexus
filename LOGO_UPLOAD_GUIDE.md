# 📸 Руководство по загрузке логотипов сервисов

## Для владельцев (max.sokolvp@gmail.com, belovodvadim@gmail.com)

### Как загрузить логотип при создании сервиса

1. **Откройте страницу "Управление подписками"** (Subscriptions)
2. **Нажмите кнопку "Добавить сервис"**
3. **Заполните основную информацию:**
   - Название сервиса
   - Категория
   - Описание

4. **Загрузка логотипа:**
   - Нажмите на кнопку "Загрузить иконку" (Upload Icon)
   - Выберите файл с компьютера (PNG, JPG, WEBP)
   - Файл будет автоматически преобразован в base64 и сохранён

5. **Добавьте тарифные планы** (минимум 1)
6. **Нажмите "Добавить сервис"**

### Рекомендации по логотипам

#### ✅ Рекомендуется:
- **Формат:** PNG с прозрачным фоном
- **Размер:** 256x256px или 512x512px (квадратный)
- **Вес файла:** До 200 КБ
- **Качество:** Чёткое, без размытия
- **Стиль:** Официальный логотип компании

#### ❌ Не рекомендуется:
- Скриншоты веб-сайтов
- Растянутые или сжатые изображения
- Фотографии низкого качества
- Слишком большие файлы (>500 КБ)

### Примеры хороших логотипов

```
✅ Netflix - квадратный красный логотип на прозрачном фоне
✅ Spotify - зелёный круг с белым логотипом
✅ YouTube - красная иконка play в белом прямоугольнике
✅ Apple Music - красно-белый градиент с нотой
```

### Что происходит с логотипом

1. **При создании сервиса:**
   - Файл преобразуется в base64
   - Сохраняется в поле `services.icon`
   - Отображается в списке сервисов

2. **При выборе пользователем:**
   - Логотип копируется в `subscriptions.logo`
   - Отображается в Dashboard, Analytics, списке подписок

3. **Fallback система:**
   - Если логотип не загружается → показывается первая буква названия
   - Буква на градиентном фоне (фиолетово-оранжевый)

### Редактирование логотипа

1. Откройте страницу "Управление подписками"
2. Найдите нужный сервис в таблице
3. Нажмите кнопку "Редактировать" (иконка карандаша)
4. Загрузите новый логотип
5. Сохраните изменения

**⚠️ Внимание:** Обновление логотипа сервиса НЕ влияет на уже созданные подписки пользователей. Логотип копируется в момент создания подписки.

### Технические детали

#### Где хранятся логотипы:
- **Таблица:** `services`
- **Поле:** `icon` (text)
- **Формат:** base64 encoded string или URL

#### Как отображаются:
```tsx
<ServiceLogo 
  logo={service.icon}  // URL или base64
  name={service.name}  // Для fallback
  size="md"            // sm, md, lg, xl
/>
```

#### Автоматический fallback:
- Компонент `ServiceLogo` автоматически определяет ошибки загрузки
- При ошибке показывает первую букву названия
- Градиентный фон адаптируется под тему (светлая/тёмная)

### Частые вопросы

**Q: Можно ли использовать URL вместо загрузки файла?**  
A: Да, можно вставить URL логотипа вручную. Важно, чтобы URL был публичным и доступным.

**Q: Почему логотип не отображается?**  
A: Возможные причины:
- Неверный URL
- Файл слишком большой
- Формат не поддерживается браузером
- Проблема с кодировкой base64

**Q: Как изменить логотип для уже существующих подписок?**  
A: Логотип в подписках независим от логотипа в сервисе. Нужно попросить пользователей пересоздать подписку или обновить вручную через БД.

**Q: Можно ли загрузить SVG?**  
A: Технически да, но рекомендуем PNG для лучшей совместимости.
