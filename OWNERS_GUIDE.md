# Руководство для владельцев SubManager

## Роли и права доступа

### Владельцы (Owners)
Владельцами приложения являются следующие пользователи:
- **max.sokolvp@gmail.com** 
- **belovodvadim@gmail.com**

Владельцы имеют полные права на управление сервисами:
- ✅ Создание новых сервисов
- ✅ Редактирование существующих сервисов
- ✅ Удаление сервисов
- ✅ Управление тарифными планами

### Обычные пользователи
Обычные пользователи могут:
- ✅ Просматривать все созданные сервисы
- ✅ Добавлять подписки из списка сервисов
- ✅ Редактировать свои подписки
- ✅ Удалять свои подписки
- ❌ Создавать/редактировать/удалять сервисы

## Управление сервисами (для владельцев)

### Вкладка "Подписки" для владельцев

Когда владелец заходит во вкладку "Подписки", вместо личных подписок он видит **панель управления сервисами**.

#### Создание нового сервиса

1. Нажмите кнопку **"Добавить сервис"**
2. Заполните обязательные поля:
   - **Название сервиса** - например, "Netflix", "Spotify"
   - **Категория** - выберите из списка:
     - Кино и сериалы
     - Музыка
     - Мульти подписки
     - Здоровье
     - Интернет и телеком
     - Бизнес и маркетинг
     - Доставка
     - Социальные сети
     - Облачные хранилища
     - Кибербезопасность
     - Книги
     - Игры и стриминг
     - Разработка и дизайн
     - Образование
     - Финансы

3. Заполните дополнительные поля (опционально):
   - **Иконка сервиса** - загрузите логотип сервиса
   - **Краткое описание** - опишите сервис
   - **Популярный сервис** - отметьте, если сервис популярный (будет помечен звездочкой)

4. Добавьте **тарифные планы**:
   - **Название тарифа** - "Базовый", "Премиум", "Семейный" и т.д.
   - **Цена** - стоимость тарифа
   - **Валюта** - RUB, USD, EUR, CNY, BYN
   - **Период** - Месяц или Год
   - Можно добавить несколько тарифов для одного сервиса

5. Нажмите **"Добавить сервис"**

#### Редактирование сервиса

1. В таблице сервисов найдите нужный сервис
2. Нажмите на иконку **карандаша** (Редактировать)
3. Измените необходимые поля
4. Нажмите **"Сохранить изменения"**

#### Удаление сервиса

⚠️ **ВАЖНО**: При удалении сервиса автоматически удаляются **все подписки пользователей** на этот сервис!

1. В таблице сервисов найдите нужный сервис
2. Нажмите на иконку **корзины** (Удалить)
3. Подтвердите удаление во всплывающем окне
4. Сервис и все связанные подписки пользователей будут удалены

#### Фильтрация сервисов

Используйте кнопки фильтрации по категориям для быстрого поиска нужных сервисов.

## Процесс добавления подписки (для обычных пользователей)

### Новый процесс

Теперь обычные пользователи не могут создавать сервисы вручную. Вместо этого:

1. Пользователь нажимает **"Добавить подписку"**
2. Открывается модальное окно **"Выберите сервис"** со списком всех сервисов, созданных владельцами
3. Пользователь может:
   - Искать сервисы по названию или описанию
   - Фильтровать по категориям
   - Видеть популярные сервисы (отмечены звездочкой)
4. После выбора сервиса:
   - Пользователь выбирает подходящий тариф
   - Указывает дату следующего платежа
   - Нажимает "Добавить подписку"
5. Подписка добавляется в список пользователя

## База данных Supabase

### Таблица `services`

Для работы системы необходимо создать таблицу `services` в Supabase:

1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Выполните SQL-скрипт из файла `supabase-services-table.sql`

### Основные особенности:

- **Row Level Security (RLS)** включен
- Все аутентифицированные пользователи могут **просматривать** сервисы
- Только владельцы могут **создавать, редактировать и удалять** сервисы
- При удалении сервиса автоматически удаляются все связанные подписки (trigger)

### Политики безопасности:

```sql
-- Просмотр: все пользователи
CREATE POLICY "Anyone can view services"
    ON public.services FOR SELECT
    TO authenticated USING (true);

-- Создание: только владельцы
CREATE POLICY "Only owners can insert services"
    ON public.services FOR INSERT
    TO authenticated
    WITH CHECK (auth.jwt() ->> 'email' IN ('max.sokolvp@gmail.com', 'belovodvadim@gmail.com'));

-- Редактирование: только владельцы
CREATE POLICY "Only owners can update services"
    ON public.services FOR UPDATE
    TO authenticated
    USING (auth.jwt() ->> 'email' IN ('max.sokolvp@gmail.com', 'belovodvadim@gmail.com'));

-- Удаление: только владельцы
CREATE POLICY "Only owners can delete services"
    ON public.services FOR DELETE
    TO authenticated
    USING (auth.jwt() ->> 'email' IN ('max.sokolvp@gmail.com', 'belovodvadim@gmail.com'));
```

```sql
    )
)
```

## Troubleshooting

### Проблема: Владелец не может создавать сервисы

**Решение:**
1. Проверьте, что вы вошли с правильным email (max.sokolvp@gmail.com или belovodvadim@gmail.com)
2. Проверьте, что политики RLS правильно настроены в Supabase
3. Проверьте консоль браузера на наличие ошибок

### Проблема: Обычные пользователи видят пустой список сервисов

**Решение:**
1. Убедитесь, что владельцы создали хотя бы один сервис
2. Проверьте политику "Anyone can view services" в Supabase
3. Проверьте консоль браузера на наличие ошибок

### Проблема: При удалении сервиса подписки пользователей не удаляются

**Решение:**
1. Проверьте, что trigger `cascade_delete_service_subscriptions` создан в Supabase
2. Проверьте логи в Supabase Dashboard

## Рекомендации

1. **Регулярно обновляйте сервисы**: Добавляйте новые популярные сервисы, которые могут понадобиться пользователям
2. **Проверяйте тарифы**: Актуализируйте цены при изменении стоимости сервисов
3. **Категоризация**: Правильно выбирайте категории для удобства поиска
4. **Иконки**: Загружайте качественные логотипы сервисов для лучшего UX
5. **Описания**: Добавляйте краткие, но информативные описания
6. **Осторожность при удалении**: Помните, что удаление сервиса затронет всех пользователей, которые его используют
