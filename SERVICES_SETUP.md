# Настройка системы сервисов SubManager

## Обзор
Система сервисов позволяет владельцу аккаунта (max.sokolvp@gmail.com) создавать шаблоны сервисов подписок, из которых обычные пользователи могут выбирать при добавлении своих подписок.

## Роли пользователей

### Владелец (max.sokolvp@gmail.com)
- ✅ Может создавать новые сервисы с тарифами
- ✅ Может редактировать и удалять сервисы
- ✅ Видит кнопку "Добавить подписку" в верхней панели (для создания сервисов)
- ❌ НЕ видит кнопку "Добавить подписку" на дашборде

### Обычные пользователи
- ✅ Могут выбирать из списка созданных сервисов
- ✅ Видят кнопку "Добавить подписку" на дашборде
- ❌ НЕ могут создавать свои собственные сервисы
- ❌ НЕ видят кнопку "Добавить подписку" в верхней панели

## Настройка базы данных

### Шаг 1: Создание таблицы services
Выполните SQL-скрипт из файла `supabase-services-table.sql` в вашей Supabase базе данных:

1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Скопируйте и выполните содержимое файла `supabase-services-table.sql`

### Шаг 2: Проверка RLS политик
Убедитесь, что Row Level Security (RLS) включен и политики созданы:

- **SELECT**: Все аутентифицированные пользователи могут просматривать сервисы
- **INSERT**: Только max.sokolvp@gmail.com может создавать сервисы
- **UPDATE**: Только max.sokolvp@gmail.com может обновлять сервисы
- **DELETE**: Только max.sokolvp@gmail.com может удалять сервисы

## Структура данных

### Таблица `services`
```sql
- id: UUID (primary key)
- user_id: UUID (references auth.users)
- name: TEXT (название сервиса)
- category: TEXT (категория)
- description: TEXT (описание)
- icon: TEXT (иконка в формате base64 или URL)
- is_popular: BOOLEAN (популярный сервис)
- pricing_plans: JSONB (массив тарифных планов)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

### Формат pricing_plans (JSONB)
```json
[
  {
    "id": "unique-id",
    "name": "Базовый",
    "price": 299,
    "currency": "RUB",
    "billingCycle": "monthly"
  },
  {
    "id": "unique-id-2",
    "name": "Премиум",
    "price": 599,
    "currency": "RUB",
    "billingCycle": "monthly"
  }
]
```

## Использование

### Для владельца
1. Войдите под аккаунтом max.sokolvp@gmail.com
2. Нажмите кнопку "Добавить подписку" в верхней панели
3. Заполните информацию о сервисе:
   - Название сервиса
   - Категория
   - Описание (опционально)
   - Иконка (опционально)
   - Популярность
   - Тарифные планы (минимум один)
4. Нажмите "Добавить сервис"

### Для обычных пользователей
1. На дашборде нажмите "Добавить подписку"
2. Используйте поиск и фильтры для поиска нужного сервиса
3. Выберите сервис из списка
4. Выберите тарифный план
5. Укажите дату следующего платежа
6. Нажмите "Добавить подписку"

## Особенности

### Автоматическая конвертация валют
Когда пользователь выбирает сервис с тарифом в другой валюте, цена автоматически конвертируется в валюту пользователя согласно настройкам.

### Realtime синхронизация
Добавление новых сервисов владельцем автоматически отображается у всех пользователей благодаря Supabase Realtime.

### Поиск и фильтрация
Пользователи могут искать сервисы по названию и описанию, а также фильтровать по категориям.

## Устранение неполадок

### Пользователь не видит сервисы
1. Проверьте, что таблица `services` создана
2. Проверьте RLS политики
3. Убедитесь, что пользователь аутентифицирован

### Владелец не может создать сервис
1. Проверьте, что email владельца точно `max.sokolvp@gmail.com`
2. Проверьте RLS политику для INSERT
3. Проверьте консоль браузера на наличие ошибок

### Сервисы не отображаются в списке
1. Проверьте, что сервисы созданы в базе данных
2. Проверьте формат данных в таблице
3. Проверьте консоль браузера на наличие ошибок загрузки
