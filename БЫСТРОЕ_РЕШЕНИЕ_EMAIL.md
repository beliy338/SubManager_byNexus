# Быстрое решение ошибки "Email not confirmed"

## 🔴 Проблема
При входе в систему появляется ошибка: **"Email not confirmed"**

## ✅ Быстрое решение (2 минуты)

### Шаг 1: Откройте Supabase
1. Перейдите на https://app.supabase.com
2. Выберите ваш проект **SubManager**

### Шаг 2: Отключите верификацию email
1. В боковом меню выберите **Authentication**
2. Перейдите на вкладку **Settings** или **Email Auth**
3. Найдите опцию **"Enable email confirmations"** или **"Confirm email"**
4. **ОТКЛЮЧИТЕ** эту опцию (снимите галочку)
5. Нажмите **Save** / **Сохранить**

### Шаг 3: Готово! 🎉
Теперь можно входить в систему без подтверждения email.

---

## 🔄 Альтернативный метод: Подтвердить существующих пользователей

Если у вас уже есть зарегистрированные пользователи:

1. **Authentication** → **Users**
2. Найдите нужного пользователя
3. Кликните на него
4. Поставьте галочку **Email Confirmed**
5. Сохраните

## 📧 Как это работает для пользователей

### Если верификация включена:

**При регистрации:**
1. Пользователь регистрируется
2. Видит сообщение: *"Аккаунт успешно создан! Проверьте вашу почту для подтверждения email"*
3. Получает письмо (если настроен SMTP)
4. Подтверждает email по ссылке
5. Может войти в систему

**При попытке войти с неподтвержденным email:**
1. Видит сообщение: *"Email не подтверждён. Проверьте вашу почту"*
2. Может нажать **"Отправить письмо повторно"**
3. Получит новое письмо с подтверждением

### Если верификация отключена:
1. Пользователь регистрируется
2. Может сразу войти в систему
3. Никаких писем не требуется

## ❓ FAQ

**В:** Я отключил верификацию, но ошибка осталась  
**О:** Проверьте, что вы сохранили изменения и перезапустили приложение

**В:** У меня уже есть пользователи с неподтвержденным email  
**О:** Вручную подтвердите их через Authentication → Users

**В:** Хочу использовать email верификацию  
**О:** Настройте SMTP в Project Settings → Auth → SMTP Settings

**В:** Письма не приходят  
**О:** Проверьте настройки SMTP и папку "Спам"
