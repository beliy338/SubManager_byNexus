## ❓ Быстрые ответы

### Q: Модальное окно не появляется?

```sql
-- Проверьте в Supabase SQL Editor:
SELECT ai_permission FROM user_settings WHERE user_id = 'ваш-id';

-- Установите NULL для теста:
UPDATE user_settings SET ai_permission = NULL WHERE user_id = 'ваш-id';
```

### Q: Кнопка "Анализировать" не работает?

Убедитесь, что:
1. У вас есть хотя бы одна подписка
2. AI активирован в модальном окне (ai_permission = 'yes')
3. Вы на странице Аналитики

### Q: Медленно работает?

TensorFlow.js загружается при первом использовании. Последующие запуски будут быстрее.

### Q: Можно ли добавить свои сервисы для распознавания?

Да! Откройте `/src/app/utils/tensorflow.ts` и добавьте паттерны в массив `SERVICE_PATTERNS`:

```typescript
{ pattern: /мой-сервис/i, name: 'Мой Сервис', category: 'Категория' }
```

### Q: Как добавить альтернативы для моих сервисов?

В файле `/src/app/utils/tensorflow.ts` найдите `ALTERNATIVES_DB` и добавьте:

```typescript
'Мой Сервис': [
  {
    name: 'Альтернатива',
    price: 299,
    pros: ['Плюс 1', 'Плюс 2'],
    cons: ['Минус 1'],
    savingsPercent: 30,
  }
]
```

## 📖 Дополнительная информация

### Производительность

TensorFlow.js использует:
- **WebGL** - для ускорения вычислений на GPU
- **WASM** - для быстрых вычислений на CPU
- **Автоматический выбор** лучшего бэкенда

Проверить бэкенд:
```javascript
// В консоли браузера (F12)
console.log(tf.getBackend()); // 'webgl' или 'wasm'
```

### Память

TensorFlow.js автоматически управляет памятью. После анализа тензоры очищаются.
