# Исправление аутентификации Supabase

## Проблема
Приложение выдавало ошибку "Failed to fetch" при попытке регистрации новых пользователей. Проблема была в том, что код пытался обращаться к Edge Functions (серверным функциям) Supabase, которые не были развернуты.

## Решение

### 1. Регистрация через Supabase Auth API
Изменили процесс регистрации в `/src/app/pages/Signup.tsx`:
- **Раньше**: Приложение делало запрос к Edge Function `/signup`
- **Теперь**: Используется прямой вызов `supabase.auth.signUp()` из клиентской библиотеки

```typescript
// Новый код регистрации
const { data, error: signUpError } = await supabase.auth.signUp({
  email,
  password,
  options: {
    data: {
      name: name
    }
  }
});
```

### 2. Хранение данных в localStorage
Изменили `/src/app/contexts/AppContext.tsx` для работы с локальным хранилищем:
- Все данные (подписки, настройки) временно сохраняются в `localStorage`
- API вызовы к Edge Functions закомментированы до момента их развертывания
- Данные привязываются к `user.id` из Supabase Auth

```typescript
// Пример сохранения подписок
localStorage.setItem(`subscriptions_${user.id}`, JSON.stringify(updatedSubs));
```

### 3. Автоматический вход после регистрации
После успешной регистрации пользователь автоматически входит в систему:
- Supabase возвращает сессию в ответе на `signUp()`
- AppContext отслеживает изменения auth state через `onAuthStateChange()`
- Пользователь перенаправляется на главную страницу

## Следующие шаги

Когда будут готовы к развертыванию Edge Functions:

1. **Развернуть Edge Functions в Supabase:**
   - Код функций находится в `/supabase/functions/server/`
   - Используется Hono framework для обработки запросов
   - KV store для хранения данных пользователей

2. **Раскомментировать API вызовы в AppContext.tsx:**
   - В функции `loadUserData()` - строки 92-118
   - В функции `updateSettings()` - строки 268-277
   - В функциях `addSubscription()`, `updateSubscription()`, `deleteSubscription()` - соответствующие блоки

3. **Добавить миграцию данных:**
   - При первом входе после развертывания функций
   - Перенести данные из localStorage в облако
   - Очистить локальное хранилище после успешной миграции

## Преимущества текущего подхода

1. **Приложение полностью функционально** без необходимости развертывания серверной части
2. **Используется реальная аутентификация Supabase** (не тестовые аккаунты)
3. **Легко масштабировать** - достаточно раскомментировать код и развернуть функции
4. **Безопасность** - пользовательские пароли хранятся в Supabase Auth
5. **Простая миграция** на облачное хранилище в будущем
