# ☁️ Онлайн сохранение данных в Supabase

## ✅ Реализовано

Приложение SubManager теперь полностью интегрировано с Supabase Database для хранения:
- 📝 **Подписок** - все ваши подписки синхронизируются в облаке
- ⚙️ **Настроек** - язык, валюта, тема и размер текста
- 🔐 **Аутентификации** - безопасный вход через Supabase Auth

## 🚀 Как активировать

### Шаг 1: Применить SQL миграцию

Откройте [Supabase Dashboard](https://app.supabase.com/project/echbyusirwoojodjhhzi/sql/new) и выполните:

1. Перейдите в раздел **SQL Editor**
2. Нажмите **New Query**
3. Скопируйте содержимое файла `/supabase/migrations/001_initial_schema.sql`
4. Вставьте в редактор и нажмите **Run**

**Или используйте краткую инструкцию из файла `/supabase/SETUP_DATABASE.md`**

### Шаг 2: Проверка

После применения миграции:

1. Зарегистрируйте нового пользователя или войдите существующим
2. Добавьте подписку
3. В боковом меню увидите статус **"Cloud Synced"** с зеленой иконкой ☁️
4. Данные автоматически сохраняются в Supabase

## 📊 Структура базы данных

### Таблица `subscriptions`

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Уникальный идентификатор |
| user_id | UUID | ID пользователя (внешний ключ) |
| name | TEXT | Название сервиса |

| price | DECIMAL | Стоимость |

| status | TEXT | Статус (active/inactive) |
| created_at | TIMESTAMPTZ | Дата создания |
| updated_at | TIMESTAMPTZ | Дата обновления |

| Поле | Тип | Описание |
|------|-----|----------|

## 🔒 Безопасность (Row Level Security)

Включена защита на уровне строк (RLS):

✅ Пользователи видят только свои подписки  
✅ Пользователи могут изменять только свои данные  
✅ Автоматическая фильтрация по `user_id`  
✅ Защита от SQL-инъекций  

## 🎯 Как это работает

### Регистрация
1. Пользователь создает аккаунт → сохраняется в `auth.users`
2. Автоматически создаются настройки по умолчанию в `user_settings`
3. Пользователь сразу может начать работу

### Добавление подписки
1. Пользователь заполняет форму
2. Данные отправляются в Supabase через `supabase.from('subscriptions').insert()`
3. Моментально появляются в интерфейсе
4. Индикатор показывает "Syncing..." → "Cloud Synced"

### Синхронизация между устройствами
1. Войдите на другом устройстве с тем же email/паролем
2. Все подписки автоматически загрузятся
3. Изменения на одном устройстве видны на всех

## 🔄 Индикаторы синхронизации

В боковом меню отображается статус:

- 🔄 **Syncing...** (синяя иконка) - идет сохранение
- ☁️ **Cloud Synced** (зеленая иконка) - данные в облаке
- 📱 **Local Storage** (оранжевая иконка) - только локально (демо-аккаунт)

## 🆚 Сравнение: до и после

### До (localStorage)
❌ Данные только в браузере  
❌ Нет синхронизации между устройствами  
❌ При очистке кэша все теряется  

### После (Supabase)
✅ Данные в облаке  
✅ Автоматическая синхронизация  
✅ Доступ с любого устройства  
✅ Резервное копирование  
✅ Безопасное хранение  

## 💾 Миграция существующих данных

Если у вас уже есть данные в localStorage:

1. Войдите в аккаунт
2. Данные автоматически останутся доступны локально
3. Новые подписки будут сохраняться в облако
4. Старые подписки можно вручную пересоздать в облаке

**Автоматическая миграция будет добавлена в следующей версии**

## 🐛 Решение проблем

### Ошибка "relation subscriptions does not exist"
**Причина:** SQL миграция не применена  
**Решение:** Выполните шаг 1 (применить миграцию)

### Подписки не синхронизируются
**Причина:** Политики RLS не созданы  
**Решение:** Проверьте, что миграция выполнена полностью

### Статус "Local Storage" вместо "Cloud Synced"
**Причина:** Используется демо-аккаунт  
**Решение:** Создайте реальный аккаунт через регистрацию
