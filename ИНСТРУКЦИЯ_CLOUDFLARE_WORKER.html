⚡ Cloudflare Worker

Пошаговая инструкция по развертыванию

❌ Проблема

При попытке добавить код через Dashboard Cloudflare появляется ошибка:

{ "error": "requested path is invalid" }
💡 Почему это происходит?

Эта ошибка появляется при попытке вставить код через веб-интерфейс Cloudflare Dashboard из-за неправильного формата кода, лишних символов или проблем с путями.

1

Через Wrangler CLI

✅ РЕКОМЕНДУЕТСЯ
Это самый простой и надёжный способ! Никаких ошибок с копированием кода.

1 Установите Wrangler CLI

Откройте терминал и выполните:

npm install -g wrangler

или если используете pnpm:

pnpm add -g wrangler

2 Войдите в аккаунт Cloudflare

wrangler login
Откроется браузер, где нужно авторизоваться в вашем аккаунте Cloudflare.

3 Перейдите в папку worker

cd cloudflare-worker

4 Разверните Worker

wrangler deploy
✅ После успешного развертывания вы получите URL вашего worker:
https://submanager-supabase-proxy.your-subdomain.workers.dev

5 Добавьте URL в приложение

Скопируйте полученный URL и добавьте в файл .env.production:

VITE_SUPABASE_PROXY_URL=https://submanager-supabase-proxy.your-subdomain.workers.dev
2

Через Dashboard

⚠️ ЕСЛИ НЕТ ДОСТУПА К CLI
⚠️ Внимание: Этот метод менее надёжен и может привести к ошибкам при копировании.

1 Создайте Worker в Dashboard

  • Перейдите на https://dash.cloudflare.com/
  • Откройте Workers & Pages
  • Нажмите Create Application
  • Выберите Create Worker
  • Назовите Worker: submanager-supabase-proxy

2 Скопируйте правильный код

ВАЖНО: Используйте файл cloudflare-worker-dashboard-copy.js - это специальная версия без комментариев!

Откройте файл cloudflare-worker-dashboard-copy.js и скопируйте ВЕСЬ код целиком.

3 Вставьте код

  • В редакторе Cloudflare удалите весь существующий код
  • Вставьте скопированный код
  • Нажмите Save and Deploy
Проверьте:
  • ✅ Скопирован весь код от начала до конца
  • ✅ Нет лишних пробелов в начале или конце
  • ✅ Код начинается с const SUPABASE_URL
  • ✅ Код заканчивается на };

4 Скопируйте URL Worker

После успешного развертывания скопируйте URL вашего worker и добавьте в .env.production:

VITE_SUPABASE_PROXY_URL=https://submanager-supabase-proxy.your-subdomain.workers.dev

✅ Проверка работы Worker

Тест через curl

curl https://ваш-worker-url.workers.dev/rest/v1/
Вы должны получить JSON ответ от Supabase API. Если получили - всё работает!

Просмотр логов (если есть Wrangler)

wrangler tail

Это покажет логи в реальном времени.

🔧 Решение проблем

1. "requested path is invalid"

Причина: Неправильный формат кода или лишние символы при копировании

Решение:
  • Используйте файл cloudflare-worker-dashboard-copy.js
  • Убедитесь что скопирован весь код без обрезаний
  • Лучше используйте Wrangler CLI (Метод 1)

2. CORS ошибки

Причина: Неправильные CORS заголовки

Решение:
  • Проверьте что в коде есть блок CORS_HEADERS
  • Убедитесь что 'Access-Control-Allow-Origin' установлен в '*'

3. "Worker threw exception"

Причина: Синтаксическая ошибка в коде

Решение:
  • Проверьте логи через wrangler tail
  • Убедитесь что скопирован корректный код
  • Пересоздайте Worker с нуля

4. Worker не отвечает

Причина: Worker не развернут или удалён

Решение:
  • Проверьте список: wrangler deployments list
  • Проверьте в Dashboard что Worker активен
  • Попробуйте передеплоить: wrangler deploy
