Пошаговая инструкция по развертыванию

🎯 Проблема решена!

Ошибка requested path is invalid возникала из-за использования ES6 модулей. Теперь и��пользуем правильный синтаксис.

1 Откройте Cloudflare Dashboard

Перейдите по ссылке:

https://dash.cloudflare.com/

Войдите в свой аккаунт Cloudflare (или создайте новый)

2 Создайте новый Worker
  • В левом меню выберите Workers & Pages
  • Нажмите кнопку Create Application
  • Выберите Create Worker
  • Введите название: submanager-supabase-proxy
  • Нажмите Deploy (код пока не редактируем!)
💡 Совет: Можете выбрать любое название, но используйте только буквы, цифры и дефисы.
3 Откройте редактор кода
  • После создания Worker нажмите Quick Edit
  • Выделите весь существующий код
  • Удалите его (Ctrl+A, Delete)
4 Скопируйте правильный код

Откройте файл в вашем проекте:

CLOUDFLARE_DASHBOARD_SIMPLE.js
⚠️ ВАЖНО!

Используйте именно этот файл: CLOUDFLARE_DASHBOARD_SIMPLE.js

НЕ используйте файл cloudflare-worker-dashboard-copy.js - он устарел!

  • Откройте файл CLOUDFLARE_DASHBOARD_SIMPLE.js
  • Скопируйте весь код (Ctrl+A, Ctrl+C)
  • Вернитесь в Cloudflare Dashboard
  • Вставьте код в редактор (Ctrl+V)
5 Сохраните и разверните
  • Нажмите кнопку Save and Deploy
  • Дождитесь завершения деплоя (обычно 5-10 секунд)
✅ Поздравляем!

Ваш Worker успешно развернут!

6 Скопируйте URL Worker

После деплоя вы увидите URL вашего Worker:

https://submanager-supabase-proxy.ваш-аккаунт.workers.dev
  • Скопируйте этот URL
  • Сохраните его - он понадобится для настройки приложения
7 Проверьте работу Worker

Откройте в браузере:

https://ваш-worker.workers.dev/rest/v1/

Должен вернуться JSON от Supabase:

{"message": "OK"}
💡 Совет: Если видите JSON - всё работает отлично! ✅
8 Настройте приложение

Создайте файл .env.production в корне проекта:

VITE_SUPABASE_PROXY_URL=https://ваш-worker.workers.dev
  • Замените ваш-worker.workers.dev на ваш реальный URL
  • Сохраните файл
🎉 Готово!

Теперь ваше приложение будет работать в России без VPN через Cloudflare Worker!

❓ Часто задаваемые вопросы

Q: Почему старый файл не работал?
A: Он использовал export default, который не поддерживается в Cloudflare Dashboard. Новый файл использует addEventListener.
Q: Можно ли использовать CLI вместо Dashboard?
A: Да! См. файл CLOUDFLARE_КОМАНДЫ.md для инструкций.
Q: Worker не работает, что делать?
A: Проверьте логи в Cloudflare Dashboard → Worker → Logs. Убедитесь что скопировали весь код из правильного файла.
Q: Нужно ли платить за Worker?
A: Cloudflare предоставляет 100,000 запросов в день бесплатно. Этого более чем достаточно для личного использования.
