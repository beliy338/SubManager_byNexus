1 Регистрация на Cloudflare

Откройте в браузере:

👉 https://dash.cloudflare.com/sign-up
  • Заполните форму регистрации (Email + Пароль)
  • Нажмите Sign Up
  • Подтвердите email (проверьте почту)

⏱️ Время: 2 минуты

2 Создание Worker

После входа в Cloudflare Dashboard:

  • В левом меню найдите Workers & Pages
  • Нажмите Create application
  • Выберите вкладку Workers
  • Нажмите Create Worker
  • Название: submanager-supabase-proxy
  • Нажмите Deploy

⏱️ Время: 1 минута

3 Вставка кода

На странице созданного Worker:

  • Нажмите Edit code (или Quick Edit)
  • Выделите ВСЕ (Ctrl+A) и удалите (Delete)
  • Откройте файл cloudflare-worker-code.js из проекта
  • Скопируйте весь код из файла (Ctrl+A, Ctrl+C)
  • Вставьте в редактор Cloudflare (Ctrl+V)
  • Нажмите Save and Deploy
✅ Готово! Вы должны увидеть "Successfully deployed"

⏱️ Время: 2 минуты

4 Получение URL Worker

Найдите URL вашего Worker (он будет выглядеть так):

https://submanager-supabase-proxy.ВАША-ЧАСТЬ.workers.dev

Где найти:

  • На главной странице Worker в разделе Preview
  • Или: Settings → Triggers → Routes
⚠️ ВАЖНО: Скопируйте этот URL! Он понадобится на следующем шаге.

⏱️ Время: 30 секунд

5 Проверка работы

Откройте в браузере (замените на ваш URL):

https://submanager-supabase-proxy.ВАША-ЧАСТЬ.workers.dev/rest/v1/

Что вы должны увидеть:

  • ✅ JSON ответ от Supabase API
  • ✅ Что-то вроде: {"message":"The server is running..."}

Если видите это - Worker работает правильно!

⏱️ Время: 30 секунд

6 Настройка приложения

Создайте файл .env.production в корне проекта SubManager:

VITE_SUPABASE_PROXY_URL=https://submanager-supabase-proxy.ВАША-ЧАСТЬ.workers.dev

Замените ВАША-ЧАСТЬ на реальный URL из шага 4!

Альтернатива: Если используете хостинг с переменными окружения:

  • Vercel: Settings → Environment Variables
  • Netlify: Site settings → Environment variables
  • GitHub Pages: Actions → Secrets

Добавьте переменную:

  • Имя: VITE_SUPABASE_PROXY_URL
  • Значение: ваш URL Worker

⏱️ Время: 1 минута

7 Пересборка приложения

Вариант А: Если есть доступ к командной строке:

npm run build

Вариант Б: Если используете хостинг:

  • Vercel: Deployments → Redeploy
  • Netlify: Deploys → Trigger deploy → Deploy site
  • GitHub: Actions → Re-run workflow

⏱️ Время: 2-5 минут (зависит от платформы)

8 Проверка (БЕЗ VPN!)

Откройте приложение SubManager БЕЗ VPN и проверьте:

  • ✅ Приложение загружается
  • ✅ Можно войти в аккаунт
  • ✅ Подписки отображаются
  • ✅ Можно добавить/изменить подписки
🎉 Поздравляю! Всё работает без VPN!

Не работает?

  • Очистите кэш браузера (Ctrl+Shift+R)
  • Откройте DevTools (F12) → Console и проверьте ошибки
  • Убедитесь, что URL в .env.production правильный
  • Проверьте, что приложение пересобрано

📊 Мониторинг и статистика

Чтобы посмотреть статистику использования Worker:

  • Откройте Cloudflare Dashboard
  • Workers & Pages → Ваш Worker
  • Вкладка Analytics

Здесь вы увидите:

  • Количество запросов
  • Ошибки (если есть)
  • Производительность

Лимиты бесплатного тарифа:

  • ✅ 100,000 запросов в день
  • ✅ Неограниченный трафик
  • ✅ 10ms CPU time на запрос

Для большинства проектов этого более чем достаточно!

❓ Часто задаваемые вопросы

Q: Это безопасно?

A: Да! Worker использует только публичный ключ API, который безопасен. Вся защита данных обеспечивается Row Level Security в Supabase.

Q: Будет ли медленнее?

A: Минимально. Cloudflare имеет серверы по всему миру. Задержка обычно 10-50ms.

Q: Сколько стоит?

A: Бесплатно до 100,000 запросов/день.

Q: Можно использовать свой домен?

A: Да! В настройках Worker → Triggers → Custom Domains
