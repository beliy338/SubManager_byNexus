## Требуется выполнить SQL-миграцию

### ⚠️ ВАЖНО: Необходимо добавить новое поле в базу данных

Для работы функции автоматического переноса просроченных подписок нужно добавить поле `previous_billing` в таблицу `subscriptions`.

**Выполните следующий SQL-запрос в Supabase SQL Editor:**

```sql
-- Готовый SQL-запрос находится в файле ADD_PREVIOUS_BILLING_COLUMN.sql

ALTER TABLE subscriptions 
ADD COLUMN IF NOT EXISTS previous_billing DATE;

CREATE INDEX IF NOT EXISTS idx_subscriptions_previous_billing ON subscriptions(previous_billing);

COMMENT ON COLUMN subscriptions.previous_billing IS 'Stores the old billing date for overdue subscriptions that have been automatically moved forward by 30 days';
```

**Инструкция:**
1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Скопируйте и вставьте SQL-запрос из файла `ADD_PREVIOUS_BILLING_COLUMN.sql`
4. Нажмите "Run" для выполнения
5. Убедитесь, что запрос выполнился успешно

## Как это работает

### Автоматический перенос просроченных подписок

1. При загрузке данных пользователя система проверяет все подписки
2. Если дата `next_billing` уже прошла (дата в прошлом), система:
   - Сохраняет текущую дату `next_billing` в поле `previous_billing`
   - Добавляет 30 дней к старой дате и сохраняет в `next_billing`
3. В календаре:
   - На **старой дате** отображается подписка с **красной диагональной линией** (просроченный платеж)
   - На **новой дате** (через 30 дней) отображается та же подписка как обычный будущий платеж

### Пример

Допустим, у вас есть подписка на Netflix с датой списания 15 марта 2026:
- Сегодня 20 марта 2026, и подписка просрочена на 5 дней
- Система автоматически:
  - Сохраняет `previous_billing = 2026-03-15`
  - Устанавливает `next_billing = 2026-04-14` (15 марта + 30 дней)
- В календаре вы увидите:
  - **15 марта** - логотип Netflix с красной диагональной линией
  - **14 апреля** - логотип Netflix как обычный будущий платеж

## Дополнительная информация

- Автоматический перенос подписок происходит **только при загрузке данных** (вход в систему, обновление страницы)
- Если подписка просрочена несколько раз, каждый раз сохраняется только последняя просроченная дата
- Красная диагональная линия показывает, что платеж был пропущен
- Система не отправляет уведомления о переносе - это автоматическая фоновая операция
