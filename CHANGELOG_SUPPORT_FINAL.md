### 4. Исправление логики отправки сообщений

#### Файл: `/src/app/components/SupportModal.tsx`
**Изменения:**

1. **Исправлена логика `is_admin_reply`**:
   ```tsx
   // Было:
   is_admin_reply: userIsOwner
   
   // Стало:
   is_admin_reply: userIsOwner && replyingTo !== null
   ```
   
   **Причина**: Теперь сообщения владельца помечаются как admin_reply только когда он отвечает на конкретное сообщение пользователя.

2. **Улучшена фильтрация сообщений для владельца**:
   ```tsx
   const handleUserClick = (userId: string) => {
     const userMsgIds = messages.filter(m => m.user_id === userId).map(m => m.id);
     const userMsgs = messages.filter(msg => 
       msg.user_id === userId || 
       (msg.is_admin_reply && msg.parent_message_id && userMsgIds.includes(msg.parent_message_id))
     );
     setSelectedUserMessages(userMsgs);
   };
   ```

3. **Обновлён useEffect для синхронизации**:
   - Правильная фильтрация при обновлении сообщений
   - Включает как сообщения пользователя, так и ответы владельца

## 🔍 Исправленные баги

### ❌ Проблема #1: Пользователи не видели ответы поддержки
**Причина**: RLS политика позволяла видеть только `user_id = auth.uid()`

**Решение**: 
- Обновлена SELECT политика
- Добавлена проверка `is_admin_reply AND parent_message_id IN (...)`

---

### ❌ Проблема #2: Владельцы не видели свои сообщения в чате
**Причина**: `is_admin_reply = true` для всех сообщений владельца

**Решение**: 
- Изменено на `is_admin_reply = userIsOwner && replyingTo !== null`
- Теперь только ответы помечаются как admin_reply

---

### ❌ Проблема #3: Кнопка поддержки только на некоторых страницах
**Причина**: SupportButton был в MainLayout (не рендерился на Login/Signup)

**Решение**: 
- Перемещён в App.tsx (глобальный уровень)
- Добавлена проверка авторизации

---

### ❌ Проблема #4: Счётчик был в неправильном месте
**Причина**: `-top-1 -right-1` позиционирование

**Решение**: 
- Изменено на `-top-2 -right-2`
- Добавлен `ring-2 ring-background` для видимости
