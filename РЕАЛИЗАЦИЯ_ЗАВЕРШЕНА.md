## 🔒 Безопасность

### Row Level Security
```sql
✅ Просмотр: ВСЕ аутентифицированные пользователи
✅ Создание: ТОЛЬКО владельцы (2 человека)
✅ Редактирование: ТОЛЬКО владельцы (2 человека)
✅ Удаление: ТОЛЬКО владельцы (2 человека)
```

### Каскадное удаление
```sql
DELETE FROM services WHERE id = 'xxx'
↓
TRIGGER: cascade_delete_service_subscriptions
↓
DELETE FROM subscriptions WHERE name = 'ServiceName'
```

### Realtime синхронизация
```
Владелец создает сервис → Все видят его мгновенно
Владелец удаляет сервис → Подписки удаляются у всех
```

## 🐛 Troubleshooting

### Проблема: Владелец не может создать сервис

**Решение:**
1. Проверьте email входа
2. Проверьте RLS политики в Supabase
3. Проверьте консоль браузера (F12)

### Проблема: Подписки не удаляются при удалении сервиса

**Решение:**
1. Проверьте триггер в Supabase
2. Проверьте имя сервиса (должно совпадать)
3. Проверьте логи в Supabase

### Проблема: Обычный пользователь не видит сервисы

**Решение:**
1. Проверьте, что сервисы созданы
2. Проверьте политику "Anyone can view services"
3. Проверьте консоль браузера

**Подробнее**: См. раздел Troubleshooting в `OWNERS_GUIDE.md`
