🚀 Установка флагов стран в SubManager

Подробная инструкция по добавлению флагов в Supabase Database

📋 Шаг 1: Откройте Supabase Dashboard

  1. Перейдите на https://supabase.com
  2. Войдите в свой проект SubManager
  3. В боковом меню выберите SQL Editor
💡 Подсказка: SQL Editor находится в разделе "Database" в левом меню.

⚡ Шаг 2: Выполните SQL скрипт

  1. В SQL Editor нажмите "New query" или "+ New query"
  2. Откройте файл /supabase-country-flags-table.sql в вашем проекте
  3. Скопируйте ВЕСЬ код из файла
  4. Вставьте код в редактор Supabase
  5. Нажмите кнопку "Run" (или Ctrl/Cmd + Enter)
  6. Дождитесь сообщения ✅ Success
✅ Успех! Если вы видите "Success" - таблица создана и данные добавлены!
⚠️ Важно: Убедитесь что скопировали ВЕСЬ код, включая INSERT команды в конце файла.

🔍 Шаг 3: Проверьте результат

  1. В боковом меню Supabase выберите Table Editor
  2. Найдите таблицу country_flags в списке слева
  3. Убедитесь что в таблице есть 4 записи

🌐 Шаг 4: Проверьте флаги на сайте

  1. Откройте ваш опубликованный сайт: https://body-order-32825369.figma.site
  2. Найдите переключатель языков (иконка глобуса 🌐 + флаг)
  3. Убедитесь что флаг текущего языка отображается
  4. Кликните на переключатель чтобы открыть выпадающий список
  5. Проверьте что все 4 флага видны и отображаются корректно
✅ Отлично! Если флаги отображаются - установка прошла успешно!

✨ Что было сделано в коде

Обновлен компонент LanguageSelector

Файл: /src/app/components/LanguageSelector.tsx

  • Добавлена загрузка флагов из Supabase Database
  • Используется useEffect для автоматической загрузки
  • Резервные figma:asset флаги на случай ошибки
  • Флаги кешируются в state компонента
  • Автоматический fallback при ошибке загрузки

Логика работы:

  1. При загрузке компонента → запрос к Supabase country_flags
  2. Если успех → используются флаги из CDN (flagcdn.com)
  3. Если ошибка → используются резервные локальные флаги
  4. Флаги сохраняются в state для быстрого доступа

🎯 Преимущества решения

  • Работает везде: На любом опубликованном сайте
  • Быстро: CDN обеспечивает мгновенную загрузку
  • Безопасно: RLS политики защищают данные
  • Централизованно: Один источник для всех флагов
  • Надежно: Автоматический fallback на резервные флаги
  • Адаптивно: Работает на всех устройствах

🔧 Решение возможных проблем

❌ Проблема: SQL скрипт не выполняется

Решение:
  1. Убедитесь что скопировали ВЕСЬ код из файла
  2. Проверьте что проект Supabase активен
  3. Попробуйте выполнить скрипт по частям

❌ Проблема: Флаги не отображаются на сайте

Решение:
  1. Откройте консоль браузера (F12)
  2. Проверьте наличие ошибок
  3. Убедитесь что таблица country_flags создана
  4. Проверьте CDN: flagcdn.com/w320/ru.png

❌ Проблема: Ошибка в консоли браузера

Если видите ошибку загрузки флагов - не волнуйтесь! Приложение автоматически переключится на резервные флаги. Проверьте что:
  • Таблица country_flags существует
  • В таблице есть 4 записи
  • RLS политики настроены правильно
