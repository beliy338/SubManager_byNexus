# 🚀 Как включить работу БЕЗ VPN (5 минут)

## Что нужно сделать

Развернуть прокси-сервер на Cloudflare Workers через браузер (без командной строки!)

---

## 📝 ШАГ 1: Регистрация

👉 Откройте: **https://dash.cloudflare.com/sign-up**

- Зарегистрируйтесь (бесплатно)
- Подтвердите email

---

## 📝 ШАГ 2: Создайте Worker

👉 В Cloudflare Dashboard:

1. **Workers & Pages** (в левом меню)
2. **Create application**
3. **Create Worker**
4. Название: `submanager-supabase-proxy`
5. **Deploy**

---

## 📝 ШАГ 3: Вставьте код

👉 В созданном Worker:

1. Нажмите **Edit code**
2. **Удалите** весь существующий код (Ctrl+A, Delete)
3. Откройте файл **`cloudflare-worker-code.js`** из проекта
4. **Скопируйте** весь код (Ctrl+A, Ctrl+C)
5. **Вставьте** в редактор (Ctrl+V)
6. **Save and Deploy**

---

## 📝 ШАГ 4: Скопируйте URL

На странице Worker найдите URL:
```
https://submanager-supabase-proxy.ВАША-ЧАСТЬ.workers.dev
```

**Скопируйте его!** ✂️

---

## 📝 ШАГ 5: Настройте приложение

Создайте файл **`.env.production`** в корне проекта:

```env
VITE_SUPABASE_PROXY_URL=https://submanager-supabase-proxy.ВАША-ЧАСТЬ.workers.dev
```

(Замените на свой URL)

---

## 📝 ШАГ 6: Пересоберите приложение

**Если есть доступ к командам:**
```bash
npm run build
```

**Если нет:**
- Vercel/Netlify: сделайте Redeploy
- GitHub: запустите Actions заново

---

## ✅ Готово!

Откройте приложение **без VPN** - должно работать!

## ❌ Не работает?

1. Очистите кэш браузера (Ctrl+Shift+R)
2. Проверьте URL в `.env.production`
3. Откройте F12 → Console и посмотрите ошибки
4. Проверьте что Worker развернут: откройте `ваш-url.workers.dev/rest/v1/` в браузере
